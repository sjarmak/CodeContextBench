This enables dynamic partition assignment where the group coordinator automatically distributes partitions across group members.\n\n**Parameters:**\n- `topics`: Collection of topic names (not null)\n\n**Behavior:**\n- Can only be called once; calling again replaces the subscription\n- Must not be mixed with `assign()` calls\n- Triggers a rebalance on the next `poll()`\n- Partitions are assigned by the group coordinator\n\n**Exceptions:**\n- `IllegalArgumentException`: If topics collection is empty or null\n- `IllegalStateException`: If `assign()` was previously called\n\n**Example:**\n```java\nconsumer.subscribe(Arrays.asList(\"topic-A\", \"topic-B\"));\n```\n\n### subscribe(Collection<String> topics, ConsumerRebalanceListener listener)\n\nSubscribe to topics with callback notifications during rebalancing.\n\n**Parameters:**\n- `topics`: Collection of topic names\n- `listener`: Callback handler for partition changes\n\n**Behavior:**\n- Listener's `onPartitionsRevoked()` is called before rebalance starts\n- Listener's `onPartitionsAssigned()` is called after new assignment\n- `onPartitionsRevoked()` is a critical point to commit offsets for at-least-once semantics\n- `onPartitionsLost()` is called if rebalance fails (partition ownership lost)\n\n**Example:**\n```java\nconsumer.subscribe(\n    Arrays.asList(\"my-topic\"),\n    new ConsumerRebalanceListener() {\n        @Override\n        public void onPartitionsRevoked(Collection<TopicPartition> partitions) {\n            consumer.commitSync();\n        }\n\n        @Override\n        public void onPartitionsAssigned(Collection<TopicPartition> partitions) {\n            // Initialize position if needed\n            consumer.seekToBeginning(partitions);\n        }\n    }\n);\n```\n\n### assign(Collection<TopicPartition> partitions)\n\nManually assign partitions to this consumer. This disables automatic partition assignment through the consumer group.\n\n**Parameters:**\n- `partitions`: Collection of TopicPartition objects\n\n**Behavior:**\n- Disables the consumer group mechanism\n- Consumer is responsible for all partition management\n- Useful for standalone consumers or custom assignment logic\n- No rebalancing occurs with manual assignment\n- Replaces any previous assignment\n\n**Exceptions:**\n- `IllegalArgumentException`: If collection is empty or null\n- `IllegalStateException`: If `subscribe()` was previously called\n\n**Example:**\n```java\nTopicPartition partition0 = new TopicPartition(\"topic\", 0);\nTopicPartition partition1 = new TopicPartition(\"topic\", 1);\nconsumer.assign(Arrays.asList(partition0, partition1));\n```\n\n### subscription()\n\nGet the set of topics currently subscribed to.\n\n**Returns:** Immutable set of topic names (empty if using manual assignment)\n\n### assignment()\n\nGet the set of partitions currently assigned to this consumer.\n\n**Returns:** Immutable set of TopicPartition objects\n\n### unsubscribe()\n\nUnsubscribe from all topics and revoke all partition assignments.\n\n**Behavior:**\n- Clears the subscription\n- Revokes all assigned partitions\n- Subsequent `poll()` calls will return empty records\n- Triggers `onPartitionsRevoked()` callback if listener was set\n\n---\n\n## Polling and Data Fetching\n\n### poll(Duration timeout)\n\nFetch a batch of records and handle group coordination.\n\n**Parameters:**\n- `timeout`: Maximum time to block waiting for records (Duration.ZERO = non-blocking, Duration.ofMillis(0) for non-blocking)\n\n**Returns:** `ConsumerRecords<K, V>` containing fetched records (may be empty)\n\n**Critical Behavioral Semantics:**\n\n1. **Blocking Behavior:**\n   - Returns immediately if records are available\n   - Blocks up to `timeout` waiting for records\n   - **During rebalancing**: May block LONGER than `timeout` while executing rebalance callbacks\n   - Returns with whatever records are available after the timeout expires\n\n2. **Rebalance Handling:**\n   - Rebalances happen ONLY during `poll()` calls\n   - When a rebalance is triggered:\n     - Pending fetches are cancelled\n     - `onPartitionsRevoked()` callback is executed\n     - Group joins the rebalance\n     - `onPartitionsAssigned()` callback is executed\n     - Resume fetching\n\n3. **Group Coordination:**\n   - Sends heartbeats to group coordinator\n   - If no `poll()` calls occur within `max.poll.interval.ms`, consumer is removed from group (session timeout)\n   - Consumer automatically rejoins on next `poll()`\n\n4. **Offset Management:**\n   - If `enable.auto.commit` is enabled, auto-commits are triggered periodically during `poll()`\n\n5. **Empty Results:**\n   - Returns empty `ConsumerRecords` if:\n     - No records are available yet\n     - Timeout expires before records arrive\n     - Consumer was paused for some partitions\n\n**Exceptions:**\n- `WakeupException`: If `wakeup()` was called from another thread\n- `CommitFailedException`: If offset commit fails during rebalance\n- `RecordDeserializationException`: If message deserialization fails\n- `AuthenticationException`: If authentication fails\n\n**Example - Basic Polling Loop:**\n```java\ntry {\n    while (true) {\n        ConsumerRecords<String, String> records = consumer.poll(Duration.ofMillis(100));\n\n        for (ConsumerRecord<String, String> record : records) {\n            System.out.println(record.key() + \" -> \" + record.value());\n        }\n    }\n} finally {\n    consumer.close();\n}\n```\n\n### poll(long timeoutMillis) [Deprecated]\n\nLegacy variant of `poll(Duration)` accepting milliseconds as long.\n\n---\n\n## Offset Management\n\n### commitSync()\n\nSynchronously commit the current position for all subscribed partitions.\n\n**Behavior:**\n- Blocks until the broker acknowledges the commit\n- Commits the position (next offset to be read) for all partitions\n- If rebalance occurs during commit, throws `CommitFailedException`\n- Should be called after successfully processing records\n\n**Exceptions:**\n- `CommitFailedException`: If rebalance occurs during commit attempt\n- `WakeupException`: If `wakeup()` called while blocking\n- `IllegalStateException`: If no group is assigned (using manual assignment without group)\n\n**Example - Manual Offset Management:**\n```java\nconsumer.subscribe(Arrays.asList(\"topic\"));\nconsumer.subscribe(Arrays.asList(\"my-topic\"));\n\nwhile (true) {\n    ConsumerRecords<String, String> records = consumer.poll(Duration.ofMillis(100));\n\n    for (ConsumerRecord<String, String> record : records) {\n        processRecord(record);\n    }\n\n    // Commit after successful processing\n    try {\n        consumer.commitSync();\n    } catch (CommitFailedException e) {\n        // Handle rebalance occurred during commit\n        log.error(\"Commit failed, will retry on next poll\", e);\n    }\n}\n```\n\n### commitSync(Map<TopicPartition, OffsetAndMetadata> offsets)\n\nSynchronously commit specific offsets for specific partitions.\n\n**Parameters:**\n- `offsets`: Map of TopicPartition to OffsetAndMetadata to commit\n\n**Behavior:**\n- Allows fine-grained control over which offsets are committed\n- Only affects the specified partitions\n- Other partitions maintain their previously committed offset\n- Commits the specified offsets regardless of current position\n\n**Example:**\n```java\nMap<TopicPartition, OffsetAndMetadata> offsetsToCommit = new HashMap<>();\noffsetsToCommit.put(\n    new TopicPartition(\"topic\", 0),\n    new OffsetAndMetadata(nextOffset)\n);\nconsumer.commitSync(offsetsToCommit);\n```\n\n### commitAsync()\n\nAsynchronously commit the current position for all partitions.\n\n**Behavior:**\n- Returns immediately (non-blocking)\n- Commit happens in the background\n- Useful for high-throughput scenarios where waiting for commit is a bottleneck\n- If rebalance occurs before commit completes, commit is discarded (safe for distributed processing)\n- Does not throw exceptions on rebalance\n\n**Exceptions:**\n- `IllegalStateException`: If no group is assigned\n\n**Example:**\n```java\nfor (ConsumerRecord<String, String> record : records) {\n    processRecord(record);\n}\nconsumer.commitAsync();\n```\n\n### commitAsync(OffsetCommitCallback callback)\n\nAsynchronously commit with callback notification.\n\n**Parameters:**\n- `callback`: Called with result (success or exception) after commit completes\n\n**Behavior:**\n- Returns immediately\n- Callback is invoked when commit completes or fails\n- Callback exceptions should be handled but don't interrupt processing\n- Useful for logging or metrics\n\n**Example:**\n```java\nconsumer.commitAsync((offsets, exception) -> {\n    if (exception != null) {\n        log.error(\"Async commit failed: {}\", exception.getMessage());\n    } else {\n        log.info(\"Async commit succeeded for offsets: {}\", offsets);\n    }\n});\n```\n\n### commitAsync(Map<TopicPartition, OffsetAndMetadata> offsets, OffsetCommitCallback callback)\n\nAsynchronously commit specific offsets with callback.\n\n### position(TopicPartition partition)\n\nGet the current position (next offset to be fetched) for a partition.\n\n**Parameters:**\n- `partition`: The topic partition\n\n**Returns:** Current position (offset to be fetched next)\n\n**Behavior:**\n- Returns the offset of the next record that will be fetched\n- After fetching offset 5, position returns 6\n- Position = Last Fetched Offset + 1\n- Position only changes when `poll()` returns records or when using `seek()`\n\n**Exceptions:**\n- `IllegalStateException`: If partition is not assigned\n\n**Example:**\n```java\nconsumer.assign(Arrays.asList(new TopicPartition(\"topic\", 0)));\nconsumer.seekToBeginning(consumer.assignment());\nSystem.out.println(consumer.position(new TopicPartition(\"topic\", 0))); // Prints 0\nConsumerRecords<String, String> records = consumer.poll(Duration.ofMillis(100));\nSystem.out.println(consumer.position(new TopicPartition(\"topic\", 0))); // Prints records.count()\n```\n\n### committed(TopicPartition partition)\n\nGet the last committed offset for a partition.\n\n**Parameters:**\n- `partition`: The topic partition\n\n**Returns:** `OffsetAndMetadata` with committed offset, or null if not committed\n\n**Behavior:**\n- Returns the offset that was explicitly committed or auto-committed\n- Represents the offset the consumer can safely skip if restarting\n- Null if no offset has been committed for this partition\n- Different from `position()` - committed tracks what's been processed, position tracks what will be fetched\n\n**Critical Relationship:**\n```\nCommitted Offset represents: \"We've successfully processed and are safe to skip up to offset X\"\nPosition represents: \"The next offset we'll fetch is Y\"\nCorrect flow: Process records, then commitSync() to set committed = position\n```\n\n**Example:**\n```java\nOffsetAndMetadata committed = consumer.committed(partition);\nif (committed != null) {\n    System.out.println(\"Last committed offset: \" + committed.offset());\n} else {\n    System.out.println(\"No offset has been committed\");\n}\n```\n\n### seek(TopicPartition partition, long offset)\n\nSet the position to a specific offset in a partition.\n\n**Parameters:**\n- `partition`: The topic partition\n- `offset`: The offset to seek to (next offset to be fetched)\n\n**Behavior:**\n- Changes the position for the partition\n- Next `poll()` will fetch from the specified offset\n- Does not commit the offset\n- Useful for replaying messages or skipping to a point in time\n\n**Exceptions:**\n- `IllegalStateException`: If partition is not assigned\n\n**Example:**\n```java\nTopicPartition partition = new TopicPartition(\"topic\", 0);\nconsumer.assign(Arrays.asList(partition));\nconsumer.seek(partition, 100); // Start reading from offset 100\n```\n\n### seekToBeginning(Collection<TopicPartition> partitions)\n\nSeek to the beginning offset for partitions.\n\n**Parameters:**\n- `partitions`: Collection of topic partitions\n\n**Behavior:**\n- Sets position to 0 (or earliest available offset)\n- Does not commit\n- Useful for reprocessing all messages\n- If partitions is empty, seeks all currently assigned partitions\n\n**Example:**\n```java\nconsumer.subscribe(Arrays.asList(\"topic\"));\nconsumer.seekToBeginning(consumer.assignment()); // Reset to beginning\n```\n\n### seekToEnd(Collection<TopicPartition> partitions)\n\nSeek to the end offset for partitions.\n\n**Parameters:**\n- `partitions`: Collection of topic partitions\n\n**Behavior:**\n- Sets position to the last offset in each partition\n- Useful for skipping existing backlog and processing only new messages\n- If partitions is empty, seeks all currently assigned partitions\n- Does not commit\n\n**Example:**\n```java\nconsumer.assign(consumer.assignment());\nconsumer.seekToEnd(consumer.assignment()); // Skip all existing messages\n```\n\n### beginningOffsets(Collection<TopicPartition> partitions)\n\nGet the earliest available offset for partitions.\n\n**Parameters:**\n- `partitions`: Collection of topic partitions\n\n**Returns:** Map of TopicPartition to earliest offset\n\n**Behavior:**\n- Fetches from broker\n- Earliest offset is typically 0 (unless log compaction or retention deleted earlier messages)\n- Useful for validating offset ranges\n\n### endOffsets(Collection<TopicPartition> partitions)\n\nGet the end offset (next offset to be written) for partitions.\n\n**Parameters:**\n- `partitions`: Collection of topic partitions\n\n**Returns:** Map of TopicPartition to next offset (after last message)\n\n**Behavior:**\n- The position when seeking to end\n- Fetches from broker\n- Always >= earliest offset\n\n**Example - Offset Range Check:**\n```java\nTopicPartition tp = new TopicPartition(\"topic\", 0);\nMap<TopicPartition, Long> beginning = consumer.beginningOffsets(Arrays.asList(tp));\nMap<TopicPartition, Long> end = consumer.endOffsets(Arrays.asList(tp));\n\nlong earliestOffset = beginning.get(tp);\nlong nextOffset = end.get(tp);\nlong totalMessages = nextOffset - earliestOffset;\n\nSystem.out.println(String.format(\"Partition has %d messages (offsets %d to %d)\",\n    totalMessages, earliestOffset, nextOffset - 1));\n```\n\n### offsetsForTimes(Map<TopicPartition, Long> timestampMap)\n\nFind offsets for partitions at specified timestamps.\n\n**Parameters:**\n- `timestampMap`: Map of TopicPartition to timestamp (milliseconds)\n\n**Returns:** Map of TopicPartition to OffsetAndTimestamp (or null if no offset exists for that timestamp)\n\n**Behavior:**\n- Finds offset with timestamp >= requested timestamp\n- Returns null if timestamp is after all messages or before earliest\n- Useful for time-based seeking\n- Requires timestamp field in broker (log.message.timestamp.type)\n\n**Example - Seek to Time:**\n```java\nTopicPartition tp = new TopicPartition(\"topic\", 0);\nlong thirtyMinutesAgo = System.currentTimeMillis() - (30 * 60 * 1000);\n\nMap<TopicPartition, Long> query = new HashMap<>();\nquery.put(tp, thirtyMinutesAgo);\n\nMap<TopicPartition, OffsetAndTimestamp> results = consumer.offsetsForTimes(query);\nOffsetAndTimestamp result = results.get(tp);\n\nif (result != null) {\n    consumer.seek(tp, result.offset());\n    System.out.println(\"Seeking to offset \" + result.offset() +\n                       \" for timestamp \" + result.timestamp());\n} else {\n    System.out.println(\"No offset found for that timestamp\");\n}\n```\n\n---\n\n## Flow Control and Position Management\n\n### pause(Collection<TopicPartition> partitions)\n\nPause consumption from specified partitions.\n\n**Parameters:**\n- `partitions`: Collection of partitions to pause\n\n**Behavior:**\n- `poll()` will not fetch records from paused partitions\n- Paused partitions may still be assigned\n- Position and committed offset are not affected\n- Useful for flow control and coordinating with async processing\n\n**Example - Pausing Slow Partitions:**\n```java\nMap<TopicPartition, Long> lag = measurePartitionLag();\nfor (Map.Entry<TopicPartition, Long> entry : lag.entrySet()) {\n    if (entry.getValue() > MAX_ACCEPTABLE_LAG) {\n        consumer.pause(Arrays.asList(entry.getKey()));\n    }\n}\n```\n\n### resume(Collection<TopicPartition> partitions)\n\nResume consumption from previously paused partitions.\n\n**Parameters:**\n- `partitions`: Collection of partitions to resume\n\n**Behavior:**\n- Resumes fetching from specified partitions\n- Next `poll()` will include records from these partitions\n- No-op if partitions are not paused\n\n### paused()\n\nGet the set of paused partitions.\n\n**Returns:** Immutable set of currently paused TopicPartition objects\n\n---\n\n## Metadata and Monitoring\n\n### partitionsFor(String topic)\n\nGet partition metadata for a topic.\n\n**Parameters:**\n- `topic`: Topic name\n\n**Returns:** List of PartitionInfo for the topic (or null if topic doesn't exist)\n\n**Behavior:**\n- Fetches metadata from broker if not cached\n- May trigger metadata refresh if needed\n- Useful for understanding topic structure\n\n**Example:**\n```java\nList<PartitionInfo> partitions = consumer.partitionsFor(\"my-topic\");\nif (partitions != null) {\n    for (PartitionInfo info : partitions) {\n        System.out.println(String.format(\"Partition %d, Leader: %d\",\n            info.partition(), info.leader().id()));\n    }\n}\n```\n\n### listTopics()\n\nGet metadata for all topics the consumer has permission to access.\n\n**Returns:** Map of topic name to List<PartitionInfo>\n\n**Example:**\n```java\nMap<String, List<PartitionInfo>> topics = consumer.listTopics();\nfor (String topicName : topics.keySet()) {\n    System.out.println(\"Topic: \" + topicName);\n}\n```\n\n### metrics()\n\nGet consumer metrics.\n\n**Returns:** Map of metric name to Metric object\n\n**Behavior:**\n- Provides statistics like:\n  - `consumer-fetch-total`: Total fetch calls\n  - `records-lag-max`: Maximum lag in offsets\n  - `records-consumed-rate`: Records per second\n  - `bytes-consumed-rate`: Bytes per second\n  - `commit-latency-avg`: Average commit latency\n\n**Example:**\n```java\nMap<MetricName, ? extends Metric> metrics = consumer.metrics();\nmetrics.forEach((name, metric) -> {\n    if (name.name().contains(\"lag\")) {\n        System.out.println(name.name() + \": \" + metric.value());\n    }\n});\n```\n\n### groupId()\n\nGet the consumer group ID.\n\n**Returns:** Consumer group identifier from configuration\n\n---\n\n## Lifecycle and Resource Management\n\n### close()\n\nShutdown the consumer and release resources.\n\n**Behavior:**\n- Commits offsets if `enable.auto.commit` is true\n- Revokes partitions (calls `onPartitionsRevoked()`)\n- Leaves consumer group\n- Closes network connections\n- Must be called to clean up resources\n\n**Example:**\n```java\ntry {\n    consumer.subscribe(Arrays.asList(\"topic\"));\n    while (running) {\n        ConsumerRecords<String, String> records = consumer.poll(Duration.ofMillis(100));\n        // Process records\n    }\n} finally {\n    consumer.close(); // Always close\n}\n```\n\n### close(Duration timeout)\n\nGracefully shutdown the consumer with a timeout.\n\n**Parameters:**\n- `timeout`: Maximum time to wait for graceful shutdown\n\n**Behavior:**\n- Attempts graceful shutdown within the timeout\n- If timeout expires, forces shutdown anyway\n- Shorter than the default timeout may result in incomplete commits\n\n### wakeup()\n\nInterrupt any ongoing blocking operations in the consumer thread.\n\n**Behavior:**\n- **Thread-safe**: Can be called from any thread\n- Causes blocking operations (`poll()`, `commitSync()`, etc.) to throw `WakeupException`\n- Useful for shutting down the consumer from another thread\n- After calling `wakeup()`, the consumer thread should call `close()` to clean up\n- Multiple calls are safe; only the first takes effect until consumed\n\n**Example - Graceful Shutdown from Another Thread:**\n```java\n// Main consumer thread\nConsumerThread consumerThread = new ConsumerThread();\nconsumerThread.start();\n\n// Shutdown hook (called on JVM shutdown)\nRuntime.getRuntime().addShutdownHook(new Thread(() -> {\n    consumerThread.wakeup(); // Interrupt from another thread\n    try {\n        consumerThread.join(5000);\n    } catch (InterruptedException e) {\n        Thread.currentThread().interrupt();\n    }\n}));\n\n// In consumer thread\nclass ConsumerThread extends Thread {\n    private volatile boolean running = true;\n\n    public void wakeup() {\n        consumer.wakeup();\n        running = false;\n    }\n\n    @Override\n    public void run() {\n        try {\n            while (running) {\n                ConsumerRecords<String, String> records =\n                    consumer.poll(Duration.ofSeconds(1));\n                // Process records\n            }\n        } catch (WakeupException e) {\n            // Consumer was shut down\n        } finally {\n            consumer.close();\n        }\n    }\n}\n```\n\n---\n\n## Exception Handling\n\n### WakeupException\n\nThrown when a consumer operation is interrupted by a call to `wakeup()` from another thread.\n\n**Causes:**\n- `wakeup()` called while `poll()`, `commitSync()`, or similar blocking operation is in progress\n\n**Recovery:**\n- Catch and exit the consumer loop\n- Call `close()` to clean up resources\n- Do NOT retry immediately - the exception indicates intentional shutdown request\n\n### CommitFailedException\n\nThrown when offset commit fails due to rebalancing or other group coordination issues.\n\n**Causes:**\n- `poll()` rebalance occurred after `commitSync()` started but before completion\n- Consumer lost group membership\n- `enable.auto.commit` is true and commit failed\n\n**Recovery:**\n- If in `commitSync()`: Consumer was in rebalance, retry on next `poll()` or in rebalance callback\n- Position is unchanged; next `poll()` will refetch from current position\n\n**Example - Resilient Commit:**\n```java\npublic void commitWithRetry(KafkaConsumer<String, String> consumer, int maxRetries) {\n    int retries = 0;\n    while (retries < maxRetries) {\n        try {\n            consumer.commitSync();\n            return; // Success\n        } catch (CommitFailedException e) {\n            retries++;\n            log.warn(\"Commit failed, retry {}/{}\", retries, maxRetries);\n            if (retries >= maxRetries) {\n                // Let next poll() handle it\n                break;\n            }\n        }\n    }\n}\n```\n\n### RecordDeserializationException\n\nThrown when a message value or key cannot be deserialized.\n\n**Causes:**\n- Message bytes don't match the expected format\n- Deserializer.deserialize() throws an exception\n- Serialized object is corrupted\n\n**Behavior:**\n- Thrown from `poll()` after fetching\n- If default deserializer error handler is configured, error messages are skipped\n\n**Recovery:**\n- Catch and log the exception\n- Use a custom deserializer with error handling\n- Or use a default deserializer that logs errors\n\n**Example - Safe Deserialization:**\n```java\nProperties props = new Properties();\nprops.put(ConsumerConfig.KEY_DESERIALIZER_CLASS_CONFIG,\n    \"org.apache.kafka.common.serialization.StringDeserializer\");\nprops.put(ConsumerConfig.VALUE_DESERIALIZER_CLASS_CONFIG,\n    \"org.apache.kafka.common.serialization.StringDeserializer\");\n// Use LoggingDeserializer or other error-tolerant deserializer\n```\n\n### TimeoutException\n\nThrown when broker operations exceed configured timeouts.\n\n**Causes:**\n- Broker unresponsive\n- Network latency too high\n- `request.timeout.ms` exceeded\n\n**Tuning:**\n- Increase `request.timeout.ms` (default 30s)\n- Check broker health and network\n- Ensure `session.timeout.ms` < `request.timeout.ms`\n\n### AuthenticationException / AuthorizationException\n\nThrown when authentication or authorization fails.\n\n**Causes:**\n- Invalid credentials\n- Missing permissions for topic or group\n- SASL configuration issue\n\n**Recovery:**\n- Verify credentials and configuration\n- Check broker ACLs and permissions\n- Ensure `security.protocol` and `sasl.mechanism` match broker\n\n---\n\n## Configuration-Driven Behaviors\n\n### Critical Configuration Properties\n\n#### Consumer Group Coordination\n\n**`group.id`** (Required for group management)\n- Consumer group identifier\n- Consumers with same group ID coordinate partition assignment\n- If not set, use `assign()` for manual assignment\n\n**`max.poll.records`** (Default: 500)\n- Maximum records returned by single `poll()` call\n- Trade-off: more records per poll vs memory usage\n- Increase for high-throughput, decrease for real-time responsiveness\n\n**`max.poll.interval.ms`** (Default: 300000 = 5 minutes)\n- Maximum time between polls before removal from group\n- If `poll()` takes longer than this, consumer is removed\n- Includes processing time + network delays + rebalance callback time\n- Increase for slow message processing, but increases failover time\n\n**`session.timeout.ms`** (Default: 10000 = 10 seconds)\n- Time broker waits for heartbeat before removing from group\n- Heartbeats sent during `poll()`\n- Increase for unreliable networks, decrease for faster failover\n- Must be between `heartbeat.interval.ms` and `max.poll.interval.ms`\n\n**`heartbeat.interval.ms`** (Default: 3000)\n- Frequency of heartbeats to broker\n- Typically 1/3 of `session.timeout.ms`\n- Smaller values enable faster failure detection\n\n#### Offset Management\n\n**`enable.auto.commit`** (Default: true)\n- If true, consumer automatically commits position periodically\n- If false, application must call `commitSync()` or `commitAsync()`\n\n**`auto.commit.interval.ms`** (Default: 5000)\n- Frequency of automatic commits during `poll()`\n- Only relevant if `enable.auto.commit` is true\n\n**`auto.offset.reset`** (Default: \"latest\")\n- Action when no initial offset exists for partition:\n  - `earliest`: Start from beginning of partition\n  - `latest`: Start from end (skip existing messages)\n  - `none`: Throw exception\n\n**`isolation.level`** (Default: \"read_uncommitted\")\n- `read_uncommitted`: Fetch all messages including from uncommitted transactions\n- `read_committed`: Only fetch messages from committed transactions (up to LSO)\n\n#### Fetching Behavior\n\n**`fetch.min.bytes`** (Default: 1)\n- Broker waits until at least this many bytes available before responding\n- Increase to reduce overhead, decrease for lower latency\n\n**`fetch.max.wait.ms`** (Default: 500)\n- Broker waits maximum this long for `fetch.min.bytes` before responding\n- If 500ms expires, sends whatever bytes are available\n\n**`max.partition.fetch.bytes`** (Default: 1MB)\n- Maximum bytes fetched per partition in a single request\n- Must be less than broker's `replica.fetch.max.bytes`\n\n**`connections.max.idle.ms`** (Default: 540000)\n- Idle connections closed after this duration\n\n### Offset Commit Semantics by Configuration\n\n```\nenable.auto.commit = true, auto.commit.interval.ms = 5000:\n  - Position committed every 5 seconds during polling\n  - At-most-once semantics (may lose recent records if crash)\n\nenable.auto.commit = false + commitSync() in processing loop:\n  - Offset committed after processing each batch\n  - At-least-once semantics (may reprocess after crash)\n\nenable.auto.commit = false + commitAsync() + graceful shutdown:\n  - High throughput, eventual consistency\n  - Final commitSync() in shutdown ensures last batch\n\nisolation.level = read_committed + transactional producer:\n  - Only see committed messages + transactional semantics\n  - Slightly higher latency, consistency guaranteed\n```\n\n---\n\n## Common Patterns and Best Practices\n\n### Pattern 1: Basic Subscription with Auto-Commit\n\nFor simple use cases where message loss is acceptable:\n\n```java\nProperties props = new Properties();\nprops.put(ConsumerConfig.BOOTSTRAP_SERVERS_CONFIG, \"localhost:9092\");\nprops.put(ConsumerConfig.GROUP_ID_CONFIG, \"my-group\");\nprops.put(ConsumerConfig.KEY_DESERIALIZER_CLASS_CONFIG,\n    StringDeserializer.class);\nprops.put(ConsumerConfig.VALUE_DESERIALIZER_CLASS_CONFIG,\n    StringDeserializer.class);\nprops.put(ConsumerConfig.AUTO_OFFSET_RESET_CONFIG, callback)\n\nAsynchronously commit specific offsets with callback.\n\n### position(TopicPartition partition)\n\nGet the current position (next offset to be fetched) for a partition.\n\n**Parameters:**\n- `partition`: The topic partition\n\n**Returns:** Current position (offset to be fetched next)\n\n**Behavior:**\n- Returns the offset of the next record that will be fetched\n- After fetching offset 5, position returns 6\n- Position = Last Fetched Offset + 1\n- Position only changes when `poll()` returns records or when using `seek()`\n\n**Exceptions:**\n- `IllegalStateException`: If partition is not assigned\n\n**Example:**\n```java\nconsumer.assign(Arrays.asList(new TopicPartition(\"topic\", 0)));\nconsumer.seekToBeginning(consumer.assignment());\nSystem.out.println(consumer.position(new TopicPartition(\"topic\", 0))); // Prints 0\nConsumerRecords<String, String> records = consumer.poll(Duration.ofMillis(100));\nSystem.out.println(consumer.position(new TopicPartition(\"topic\", 0))); // Prints records.count()\n```\n\n### committed(TopicPartition partition)\n\nGet the last committed offset for a partition.\n\n**Parameters:**\n- `partition`: The topic partition\n\n**Returns:** `OffsetAndMetadata` with committed offset, or null if not committed\n\n**Behavior:**\n- Returns the offset that was explicitly committed or auto-committed\n- Represents the offset the consumer can safely skip if restarting\n- Null if no offset has been committed for this partition\n- Different from `position()` - committed tracks what's been processed, position tracks what will be fetched\n\n**Critical Relationship:**\n```\nCommitted Offset represents: \"We've successfully processed and are safe to skip up to offset X\"\nPosition represents: \"The next offset we'll fetch is Y\"\nCorrect flow: Process records, then commitSync() to set committed = position\n```\n\n**Example:**\n```java\nOffsetAndMetadata committed = consumer.committed(partition);\nif (committed != null) {\n    System.out.println(\"Last committed offset: \" + committed.offset());\n} else {\n    System.out.println(\"No offset has been committed\");\n}\n```\n\n### seek(TopicPartition partition, long offset)\n\nSet the position to a specific offset in a partition.\n\n**Parameters:**\n- `partition`: The topic partition\n- `offset`: The offset to seek to (next offset to be fetched)\n\n**Behavior:**\n- Changes the position for the partition\n- Next `poll()` will fetch from the specified offset\n- Does not commit the offset\n- Useful for replaying messages or skipping to a point in time\n\n**Exceptions:**\n- `IllegalStateException`: If partition is not assigned\n\n**Example:**\n```java\nTopicPartition partition = new TopicPartition(\"topic\", 0);\nconsumer.assign(Arrays.asList(partition));\nconsumer.seek(partition, 100); // Start reading from offset 100\n```\n\n### seekToBeginning(Collection<TopicPartition> partitions)\n\nSeek to the beginning offset for partitions.\n\n**Parameters:**\n- `partitions`: Collection of topic partitions\n\n**Behavior:**\n- Sets position to 0 (or earliest available offset)\n- Does not commit\n- Useful for reprocessing all messages\n- If partitions is empty, seeks all currently assigned partitions\n\n**Example:**\n```java\nconsumer.subscribe(Arrays.asList(\"topic\"));\nconsumer.seekToBeginning(consumer.assignment()); // Reset to beginning\n```\n\n### seekToEnd(Collection<TopicPartition> partitions)\n\nSeek to the end offset for partitions.\n\n**Parameters:**\n- `partitions`: Collection of topic partitions\n\n**Behavior:**\n- Sets position to the last offset in each partition\n- Useful for skipping existing backlog and processing only new messages\n- If partitions is empty, seeks all currently assigned partitions\n- Does not commit\n\n**Example:**\n```java\nconsumer.assign(consumer.assignment());\nconsumer.seekToEnd(consumer.assignment()); // Skip all existing messages\n```\n\n### beginningOffsets(Collection<TopicPartition> partitions)\n\nGet the earliest available offset for partitions.\n\n**Parameters:**\n- `partitions`: Collection of topic partitions\n\n**Returns:** Map of TopicPartition to earliest offset\n\n**Behavior:**\n- Fetches from broker\n- Earliest offset is typically 0 (unless log compaction or retention deleted earlier messages)\n- Useful for validating offset ranges\n\n### endOffsets(Collection<TopicPartition> partitions)\n\nGet the end offset (next offset to be written) for partitions.\n\n**Parameters:**\n- `partitions`: Collection of topic partitions\n\n**Returns:** Map of TopicPartition to next offset (after last message)\n\n**Behavior:**\n- The position when seeking to end\n- Fetches from broker\n- Always >= earliest offset\n\n**Example - Offset Range Check:**\n```java\nTopicPartition tp = new TopicPartition(\"topic\", 0);\nMap<TopicPartition, Long> beginning = consumer.beginningOffsets(Arrays.asList(tp));\nMap<TopicPartition, Long> end = consumer.endOffsets(Arrays.asList(tp));\n\nlong earliestOffset = beginning.get(tp);\nlong nextOffset = end.get(tp);\nlong totalMessages = nextOffset - earliestOffset;\n\nSystem.out.println(String.format(\"Partition has %d messages (offsets %d to %d)\",\n    totalMessages, earliestOffset, nextOffset - 1));\n```\n\n### offsetsForTimes(Map<TopicPartition, Long> timestampMap)\n\nFind offsets for partitions at specified timestamps.\n\n**Parameters:**\n- `timestampMap`: Map of TopicPartition to timestamp (milliseconds)\n\n**Returns:** Map of TopicPartition to OffsetAndTimestamp (or null if no offset exists for that timestamp)\n\n**Behavior:**\n- Finds offset with timestamp >= requested timestamp\n- Returns null if timestamp is after all messages or before earliest\n- Useful for time-based seeking\n- Requires timestamp field in broker (log.message.timestamp.type)\n\n**Example - Seek to Time:**\n```java\nTopicPartition tp = new TopicPartition(\"topic\", 0);\nlong thirtyMinutesAgo = System.currentTimeMillis() - (30 * 60 * 1000);\n\nMap<TopicPartition, Long> query = new HashMap<>();\nquery.put(tp, thirtyMinutesAgo);\n\nMap<TopicPartition, OffsetAndTimestamp> results = consumer.offsetsForTimes(query);\nOffsetAndTimestamp result = results.get(tp);\n\nif (result != null) {\n    consumer.seek(tp, result.offset());\n    System.out.println(\"Seeking to offset \" + result.offset() +\n                       \" for timestamp \" + result.timestamp());\n} else {\n    System.out.println(\"No offset found for that timestamp\");\n}\n```\n\n---\n\n## Flow Control and Position Management\n\n### pause(Collection<TopicPartition> partitions)\n\nPause consumption from is true\n- Revokes partitions (calls `onPartitionsRevoked()`)\n- Leaves consumer group\n- Closes network connections\n- Must be called to clean up resources\n\n**Example:**\n```java\ntry {\n    consumer.subscribe(Arrays.asList(\"topic\"));\n    while (running) {\n        ConsumerRecords<String, String> records = consumer.poll(Duration.ofMillis(100));\n        // Process records\n    }\n} finally {\n    consumer.close(); // Always close\n}\n```\n\n### close(Duration timeout)\n\nGracefully shutdown the consumer with a timeout.\n\n**Parameters:**\n- `timeout`: Maximum time to wait for graceful shutdown\n\n**Behavior:**\n- Attempts graceful shutdown within the timeout\n- If timeout expires, forces shutdown anyway\n- Shorter than the default timeout may result in incomplete commits\n\n### wakeup()\n\nInterrupt any ongoing blocking operations in the consumer thread.\n\n**Behavior:**\n- **Thread-safe**: Can be called from any thread\n- Causes blocking operations (`poll()`, `commitSync()`, etc.) to throw `WakeupException`\n- Useful for shutting down the consumer from another thread\n- After calling `wakeup()`, the consumer thread should call `close()` to clean up\n- Multiple calls are safe; only the first takes effect until consumed\n\n**Example - Graceful Shutdown from Another Thread:**\n```java\n// Main consumer thread\nConsumerThread consumerThread = new ConsumerThread();\nconsumerThread.start();\n\n// Shutdown hook (called on JVM shutdown)\nRuntime.getRuntime().addShutdownHook(new Thread(() -> {\n    consumerThread.wakeup(); // Interrupt from another thread\n    try {\n        consumerThread.join(5000);\n    } catch (InterruptedException e) {\n        Thread.currentThread().interrupt();\n    }\n}));\n\n// In consumer thread\nclass ConsumerThread extends Thread {\n    private volatile boolean running = true;\n\n    public void wakeup() {\n        consumer.wakeup();\n        running = false;\n    }\n\n    @Override\n    public void run() {\n        try {\n            while (running) {\n                ConsumerRecords<String, String> records =\n                    consumer.poll(Duration.ofSeconds(1));\n                // Process records\n            }\n        } catch (WakeupException e) {\n            // Consumer was shut down\n        } finally {\n            consumer.close();\n        }\n    }\n}\n```\n\n---\n\n## Exception Handling\n\n### WakeupException\n\nThrown when a consumer operation is interrupted by a call to `wakeup()` from another thread.\n\n**Causes:**\n- `wakeup()` called while `poll()`, `commitSync()`, or similar blocking operation is in progress\n\n**Recovery:**\n- Catch and exit the consumer loop\n- Call `close()` to clean up resources\n- Do NOT retry immediately - the exception indicates intentional shutdown request\n\n### CommitFailedException\n\nThrown when offset commit fails due to rebalancing or other group coordination issues.\n\n**Causes:**\n- `poll()` rebalance occurred after `commitSync()` started but before completion\n- Consumer lost group membership\n- `enable.auto.commit` is true and commit failed\n\n**Recovery:**\n- If in `commitSync()`: Consumer was in rebalance, retry on next `poll()` or in rebalance callback\n- Position is unchanged; next `poll()` will refetch from current position\n\n**Example - Resilient Commit:**\n```java\npublic void commitWithRetry(KafkaConsumer<String, String> consumer, int maxRetries) {\n    int retries = 0;\n    while (retries < maxRetries) {\n        try {\n            consumer.commitSync();\n            return; // Success\n        } catch (CommitFailedException e) {\n            retries++;\n            log.warn(\"Commit failed, retry {}/{}\", retries, maxRetries);\n            if (retries >= maxRetries) {\n                // Let next poll() handle it\n                break;\n            }\n        }\n    }\n}\n```\n\n### RecordDeserializationException\n\nThrown when a message value or key cannot be deserialized.\n\n**Causes:**\n- Message bytes don't match the expected format\n- Deserializer.deserialize() throws an exception\n- Serialized object is corrupted\n\n**Behavior:**\n- Thrown from `poll()` after fetching\n- If default deserializer error handler is configured, error messages are skipped\n\n**Recovery:**\n- Catch and log the exception\n- Use a custom deserializer with error handling\n- Or use a default deserializer that logs errors\n\n**Example - Safe Deserialization:**\n```java\nProperties props = new Properties();\nprops.put(ConsumerConfig.KEY_DESERIALIZER_CLASS_CONFIG,\n    \"org.apache.kafka.common.serialization.StringDeserializer\");\nprops.put(ConsumerConfig.VALUE_DESERIALIZER_CLASS_CONFIG,\n    \"org.apache.kafka.common.serialization.StringDeserializer\");\n// Use LoggingDeserializer or other error-tolerant deserializer\n```\n\n### TimeoutException\n\nThrown when broker operations exceed configured timeouts.\n\n**Causes:**\n- Broker unresponsive\n- Network latency too high\n- `request.timeout.ms` exceeded\n\n**Tuning:**\n- Increase `request.timeout.ms` (default 30s)\n- Check broker health and network\n- Ensure `session.timeout.ms` < `request.timeout.ms`\n\n### AuthenticationException / AuthorizationException\n\nThrown when authentication or authorization fails.\n\n**Causes:**\n- Invalid credentials\n- Missing permissions for topic or group\n- SASL configuration issue\n\n**Recovery:**\n- Verify credentials and configuration\n- Check broker ACLs and permissions\n- Ensure `security.protocol` and `sasl.mechanism` match broker\n\n---\n\n## Configuration-Driven Behaviors\n\n### Critical Configuration Properties\n\n#### Consumer Group Coordination\n\n**`group.id`** (Required for group management)\n- Consumer group identifier\n- Consumers with same group ID coordinate partition assignment\n- If not set, use `assign()` for manual assignment\n\n**`max.poll.records`** (Default: 500)\n- Maximum records returned by single `poll()` call\n- Trade-off: more records per poll vs memory usage\n- Increase for high-throughput, decrease for real-time responsiveness\n\n**`max.poll.interval.ms`** (Default: 300000 = 5 minutes)\n- Maximum time between polls before removal from group\n- If `poll()` takes longer than this, consumer is removed\n- Includes processing time + network delays + rebalance callback time\n- Increase for slow message processing, but increases failover time\n\n**`session.timeout.ms`** (Default: 10000 = 10 seconds)\n- Time broker waits for heartbeat before removing from group\n- Heartbeats sent during `poll()`\n- Increase for unreliable networks, decrease for faster failover\n- Must be between `heartbeat.interval.ms` and `max.poll.interval.ms`\n\n**`heartbeat.interval.ms`** (Default: 3000)\n- Frequency of heartbeats to broker\n- Typically 1/3 of `session.timeout.ms`\n- Smaller values enable faster failure detection\n\n#### Offset Management\n\n**`enable.auto.commit`** (Default: true)\n- If true, consumer automatically commits position periodically\n- If false, application must call `commitSync()` or `commitAsync()`\n\n**`auto.commit.interval.ms`** (Default: 5000)\n- Frequency of automatic commits during `poll()`\n- Only relevant if `enable.auto.commit` is true\n\n**`auto.offset.reset`** (Default: \"latest\")\n- Action when no initial offset exists for partition:\n  - `earliest`: Start from beginning of partition\n  - `latest`: Start from end (skip existing messages)\n  - `none`: Throw exception\n\n**`isolation.level`** (Default: \"read_uncommitted\")\n- `read_uncommitted`: Fetch all messages including from uncommitted transactions\n- `read_committed`: Only fetch messages from committed transactions (up to LSO)\n\n#### Fetching Behavior\n\n**`fetch.min.bytes`** (Default: 1)\n- Broker waits until at least this many bytes available before responding\n- Increase to reduce overhead, decrease for lower latency\n\n**`fetch.max.wait.ms`** (Default: 500)\n- Broker waits maximum this long for `fetch.min.bytes` before responding\n- If 500ms expires, sends whatever bytes are available\n\n**`max.partition.fetch.bytes`** (Default: 1MB)\n- Maximum bytes fetched per partition in a single request\n- Must be less than broker's `replica.fetch.max.bytes`\n\n**`connections.max.idle.ms`** (Default: 540000)\n- Idle connections closed after this duration\n\n### Offset Commit Semantics by Configuration\n\n```\nenable.auto.commit = true, auto.commit.interval.ms = 5000:\n  - Position committed every 5 seconds during polling\n  - At-most-once semantics (may lose recent records if crash)\n\nenable.auto.commit = false + commitSync() in processing loop:\n  - Offset committed after processing each batch\n  - At-least-once semantics (may reprocess after crash)\n\nenable.auto.commit = false + commitAsync() + graceful shutdown:\n  - High throughput, eventual consistency\n  - Final commitSync() in shutdown ensures last batch\n\nisolation.level = read_committed + transactional producer:\n  - Only see committed messages + transactional semantics\n  - Slightly higher latency, consistency guaranteed\n```\n\n---\n\n## Common Patterns and Best Practices\n\n### Pattern 1: Basic Subscription with Auto-Commit\n\nFor simple KafkaConsumer<>(props);\nconsumer.subscribe(Collections.singletonList(\"my-topic\"));\n\ntry {\n    while (true) {\n        ConsumerRecords<String, String> records =\n            consumer.poll(Duration.ofSeconds(1));\n\n        for (ConsumerRecord<String, String> record : records) {\n            try {\n                processMessage(record);\n            } catch (Exception e) RuntimeException(\"Processing failed\");\n            }\n        }\n\n        // Only commit after successful processing\n        try {\n            consumer.commitSync();\n        } catch   while (running) {\n        ConsumerRecords<String, String> records =\n            consumer.poll(Duration.ofSeconds(1));\n\n        for (ConsumerRecord<String, String> record : records) {\n Multi-Threaded Processing with Pause/Resume\n\nSingle consumer thread with worker thread pool:\n\n```java\n// Configuration\nint NUM_WORKERS = 4;\nBlockingQueue<ConsumerRecord<String, String>> queue =\n    new LinkedBlockingQueue<>(NUM_WORKERS * 2);\nExecutorService executor = Executors.newFixedThreadPool(NUM_WORKERS);\n\nKafkaConsumer<String, String> consumer = new KafkaConsumer<>(props);\nconsumer.subscribe(Collections.singletonList(\"my-topic\"));\n\n// Worker threads\nfor (int i = 0; i < NUM_WORKERS; i++) {\n    executor.submit(() -> {\n        while (true) {\n            try {\n                ConsumerRecord<String, String> record = queue.take();\n                processMessage(record);\n            } catch (InterruptedException e) {\n                Thread.currentThread().interrupt();\n                break;\n            }\n        }\n    });\n}\n\n// Consumer thread\ntry {\n    while (true) {\n        // Pause if queue is backing up\n        if (queue.size() > queue.remainingCapacity() * 0.8) {\n            consumer.pause(consumer.assignment());\n        } else if (!consumer.paused().isEmpty()) {\n            consumer.resume(consumer.paused());\n        }\n\n        ConsumerRecords<String, String> records =\n            consumer.poll(Duration.ofMillis(100));\n\n        for (ConsumerRecord<String, String> record : records) {\n            queue.put(record);\n        }\n    }\n} finally {\n    consumer.wakeup();\n    executor.shutdown();\n    executor.awaitTermination(10, TimeUnit.SECONDS);\n    consumer.close();\n}\n```\n\n**Benefits:**\n- Single consumer thread (no synchronization overhead)\n- Parallel processing (workers process while consumer fetches)\n- Flow control (pause/resume prevents queue overflow)\n\n### Pattern 6: Seeking to Time\n\nProcess messages from a specific time onwards:\n\n```java\nKafkaConsumer<String, String> consumer = new consumer.poll(Duration.ofSeconds(1));\n\n        for (ConsumerRecord<String, String> record : records) {\n            System.out.println(\"Processing: \" + record.value() +\n shutdown:\n\n```java\npublic class ConsumerRunner implements Runnable {\n    private final KafkaConsumer<String, String> consumer;\n    private volatile boolean running = true;\n\n    public ConsumerRunner(KafkaConsumer<String, String> consumer) {\n        this.consumer = consumer;\n    }\n\n    @Override\n    public void run() {\n        try {\n            consumer.subscribe(Collections.singletonList(\"my-topic\"));\n\n            while (running) {\n                try {\n                    ConsumerRecords<String, String> records =\n                        consumer.poll(Duration.ofSeconds(1));\n\n                    for (ConsumerRecord<String, String> record : records) {\n                        processMessage(record);\n                    }\n\n
