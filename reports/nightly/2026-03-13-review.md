### 1.2 HIGH: `verification_modes` Field Missing From All 274 Tasks

`configs/run_selected_tasks.sh` lines 166-169 auto-detect config mode:

```python
all_artifact_only = all(
    'direct' not in t.get('verification_modes', ['artifact'])
    for t in filtered
)
```

Zero tasks in `selected_benchmark_tasks.json` have the
`verification_modes` field. The `.get()` default of `['artifact']`
means the auto-detection always concludes every task is "artifact-only"
when all filtered tasks are MCP-unique -- because the field is
universally absent, and the default says "artifact."

**Impact**: When using `--benchmark` to filter to MCP-unique org tasks,
the script auto-switches to artifact config even when tasks support
both direct and artifact verification. Operators get unexpected config
selection with a misleading log message: "Auto-detected MCP-unique
tasks (artifact-only)."

**Fix**: Either:
1. Populate `verification_modes` for all 274 tasks based on which
   Dockerfile variants exist (e.g., tasks with both
   `Dockerfile.artifact_baseline` and `Dockerfile.artifact_full`
   get `["artifact"]`; tasks with `Dockerfile` get `["direct"]`; etc.)
2. Or change the default from `['artifact']` to `['direct', 'artifact']`
   so missing fields don't trigger artifact-only auto-detection

## 2. Feature & UX Improvements

### 2.1 No Automated Metadata Reconciliation on Commit

The task registry metadata drift (Section 1.1) has persisted across 7
nightly reports despite being "fixed" in March 7. The root cause is
that `sync_task_metadata.py --fix` only reconciles task entries but
does not update the `metadata` header block. No CI check validates
that `metadata.total_selected` matches `len(tasks)` or that
`metadata.per_suite` counts match actual per-suite task counts.

**Improvement**: Add a `metadata_integrity` check to `repo_health.py`:

```python
def check_metadata_integrity(repo_root: Path) -> int:
    path = repo_root / "configs" / "selected_benchmark_tasks.json"
    with open(path) as f:
        data = json.load(f)
    tasks = data.get("tasks", [])
    meta = data.get("metadata", {})
    errors = []
    if meta.get("total_selected") != len(tasks):
        errors.append(
            f"total_selected={meta.get('total_selected')} "
            f"but actual count={len(tasks)}"
        )
    # per-suite validation
    suite_counts = {}
    for t in tasks:
        s = t.get("benchmark", t.get("mcp_suite", ""))
        suite_counts[s] = suite_counts.get(s, 0) + 1
    for suite, count in suite_counts.items():
        meta_count = meta.get("per_suite", {}).get(suite)
        if meta_count != count:
            errors.append(f"{suite}: meta={meta_count} actual={count}")
    if errors:
        for e in errors:
            print(f"  metadata_integrity: {e}")
        return 1
    return 0
```

### 2.3 Bulk ANSWER_PATH Fix Should Be a Script

The 122-file ANSWER_PATH fix (Section 1.3) is large enough to warrant

**Improvement**: Create a migration script:

```python
#!/usr/bin/env python3
"""Migrate hardcoded ANSWER_PATH to TASK_WORKDIR fallback."""
import re
from pathlib import Path

PATTERN = re.compile(r'ANSWER_PATH="/workspace/answer\.json"')
REPLACEMENT = 'ANSWER_PATH="${ANSWER_PATH:-${TASK_WORKDIR:-/workspace}/answer.json}"'

for test_sh in Path("benchmarks").rglob("test.sh"):
    if "backups" in str(test_sh):
        continue
    text = test_sh.read_text()
    if PATTERN.search(text):
        new_text = PATTERN.sub(REPLACEMENT, text)
        test_sh.write_text(new_text)
        print(f"Fixed: {test_sh}")
```

## 3. Research Recommendations

### 3.1 Automate Task Selection Schema Validation

The missing `verification_modes` and `use_case_category` fields
indicate that `selected_benchmark_tasks.json` has no schema enforcement.
Create a JSON Schema and validate it in CI:

```json
{
  "$schema": "https://json-schema.org/draft/2020-12/schema",
  "type": "object",
  "required": ["metadata", "tasks"],
  "properties": {
    "tasks": {
      "type": "array",
      "items": {
        "type": "object",
        "required": ["task_id", "benchmark", "verification_modes"],
        "properties": {
          "verification_modes": {
            "type": "array",
            "items": {"enum": ["direct", "artifact"]},
            "minItems": 1
          },
          "use_case_category": {"type": "string"},
          "excluded": {"type": "boolean", "default": false}
        }
      }
    }
  }
}
```

This prevents future field omissions and documents the expected
structure for contributors.

### 3.2 Generate Metadata From Task Data (Don't Maintain It Separately)

The root cause of the metadata drift (Section 1.1) is that summary
metadata is maintained separately from the task entries it summarizes.
Every time tasks are added, removed, or reclassified, the metadata
must be manually updated.

Instead, compute metadata at read time:

```python
def load_task_registry(path):
    with open(path) as f:
        data = json.load(f)
    tasks = data["tasks"]
    # Compute metadata dynamically
    data["metadata"] = {
        "total_selected": len(tasks),
        "per_suite": Counter(t.get("benchmark", "") for t in tasks),
        "generated_at": datetime.utcnow().isoformat(),
    }
    return data
```

This eliminates the drift category entirely. The stored metadata
becomes advisory/historical, not authoritative.

### 3.3 Add Documentation Link Checker to CI

The broken `MCP_UNIQUE_TASKS.md` reference (Section 1.5) and the
judge dimension mismatch (Section 1.4) are documentation drift issues
that a link checker would catch automatically. Add a
`check_doc_links` function to `repo_health.py`:

```python
def check_doc_links(repo_root: Path) -> int:
    errors = 0
    for md_file in (repo_root / "docs").rglob("*.md"):
        text = md_file.read_text()
        for match in re.finditer(r'\[([^\]]+)\]\(([^)]+)\)', text):
            label, target = match.groups()
            if target.startswith("http"):
                continue  # Skip external links
            target_path = (md_file.parent / target).resolve()
            if not target_path.exists():
                print(f"  {md_file}: broken link to {target}")
                errors += 1
    return errors
```
