## Alert Evaluation Cycle Overrun Detection in grafana/grafana

**Two key files:**

### 1. `pkg/services/ngalert/schedule/schedule.go`
The main scheduler implementation:
- **`Run`** — starts the ticker, calls `schedulePeriodic`
- **`schedulePeriodic`** — the tick loop; on each tick it measures `BehindSeconds` (wall clock vs. tick time), processes the tick, and records `SchedulePeriodicDuration`
- **`processTick`** — determines which rules are ready to evaluate on the current tick
- **`runJobFn`** — **this is where overrun detection happens**: if a rule's eval channel is full (previous evaluation still running), it logs:
  ```
  Warn("Tick dropped because alert rule evaluation is too slow", ...)
  ```
  and increments the `EvaluationMissed` metric

### 2. `pkg/services/ngalert/schedule/ticker/ticker.go`
Implements a non-dropping ticker (`T`) whose `run` goroutine queues ticks without discarding them, allowing the scheduler to detect exactly how many ticks were missed when evaluation runs slow.
