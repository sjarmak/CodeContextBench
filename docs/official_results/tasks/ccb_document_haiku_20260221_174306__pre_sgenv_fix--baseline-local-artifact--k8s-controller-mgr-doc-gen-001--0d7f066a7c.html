# Kubelet Container Manager Architecture Guide

## Overview

The Kubelet Container Manager (`pkg/kubelet/cm`) is the central subsystem responsible for managing container resource constraints, allocation policies, and lifecycle integration on a Kubernetes node. It enforces quality-of-service (QoS) guarantees, manages cgroups hierarchies, coordinates resource assignments across CPU, memory, and device plugins, and ensures that pods and containers respect their requested and limit guarantees.

The container manager is initialized at kubelet startup (`pkg/kubelet/kubelet.go:316`) and serves as the authoritative interface for:
- Creating and destroying pod-level resource containers
- Enforcing node allocatable reservations
