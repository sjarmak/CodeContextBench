**Bug ID**: 207835
**Summary**: ACPI video backlight brightness control does not work on Acer TravelMate 5735Z by default

On the Intel GM45/GMA 4500MHD laptop with Debian Sid/unstable and Linux 5.6.7 and GNOME Shell 3.36.2, trying to change the brightness of the internal display using the function keys, GNOME shows the OSD, which seems to have five levels (from acer_wmi?), but the actual brightness does *not* change. Booting the system with `acpi_backlight=vendor`, exposes `/sys/devices/platform/acer-wmi/backlight/acer-wmi/brightness`, but the behavior is the same.

Booting the system with `acpi_backlight=native` or `acpi_backlight=native`, changing the brightness with the function keys works, and there now seem to be 15 (or 16) levels.

## Fault Localization Results

**Buggy File**: `drivers/acpi/video_detect.c`

**Buggy Data Structure**: `video_detect_dmi_table`

**Reasoning**: 

The Acer TravelMate 5735Z laptop's backlight only works with `acpi_backlight=native` because:

1. The laptop uses Intel GM45/GMA 4500MHD graphics with a native `intel_backlight` interface

3. The `acpi_video_get_backlight_type()` function (line 413-450) determines which backlight interface to use
4. Since there's no DMI quirk for this laptop and it's not Win8 ready, the detection falls through to `acpi_backlight_video` (line 449)

- `DMI_SYS_VENDOR` = "Acer"
- `DMI_PRODUCT_NAME` = "TravelMate 5735Z"
- Callback = `video_detect_force_native`

This is the same pattern used for other older laptops that need native backlight, such as Lenovo Ideapad S405, Dell Vostro V131, and Dell XPS 17 L702X.
