## Subsystem Overview and Purpose

The API Priority and Fairness (APF) QueueSet subsystem is the core request dispatching mechanism that implements fair queuing for server requests in Kubernetes apiserver. It prevents resource starvation by ensuring that workloads at lower priority levels cannot block higher priority workloads from making progress, and that different flow schemas within the same priority level receive fair access to execution capacity.

### Subsystem Purpose

The QueueSet subsystem solves a fundamental problem in multi-tenant API servers: how to allocate finite execution concurrency (CPU, memory, I/O) fairly across competing request flows while respecting priority levels. - **Queuing incoming requests** when the system is at capacity
