## Scheduler Framework Extension Points in kubernetes/kubernetes

### Primary Definition File
**`pkg/scheduler/framework/interface.go`** — This is the central file defining all extension point interfaces:
- `PreFilterPlugin` + `PreFilterExtensions` — pre-filter phase hook
- `FilterPlugin` — feasibility check per node
- `PostFilterPlugin` — runs when no feasible nodes (e.g. preemption)
- `PreScorePlugin` — pre-scoring state computation
- `ScorePlugin` + `ScoreExtensions` — node ranking
- `ReservePlugin` — resource reservation (and rollback)
- `PermitPlugin` — approve/deny/wait before binding
- `PreBindPlugin` — pre-binding hook
- `BindPlugin` — actual pod-to-node binding
- `PostBindPlugin` — post-binding hook
- Also: `QueueSortPlugin`, `PreEnqueuePlugin`, `EnqueueExtensions`, `Framework`, `Handle`

### Runtime Registration & Execution
**`pkg/scheduler/framework/runtime/framework.go`** — Contains `frameworkImpl` with plugin slices per extension point and `Run*Plugins` methods (e.g., `RunPreFilterPlugins`, `RunFilterPlugins`, `RunScorePlugins`, etc.)

**`pkg/scheduler/framework/runtime/instrumented_plugins.go`** — Instrumented wrappers (`instrumentedPreFilterPlugin`, `instrumentedFilterPlugin`, `instrumentedPreScorePlugin`, `instrumentedScorePlugin`) that add metrics around key extension points.
