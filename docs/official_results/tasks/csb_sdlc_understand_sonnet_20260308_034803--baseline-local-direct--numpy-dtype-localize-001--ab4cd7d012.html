# Bug Analysis: NumPy dtype Incompatibility with Pandas Nullable Integers in scikit-learn Preprocessing

## Summary

When a pandas DataFrame with nullable integer columns (`pd.Int64Dtype`, `pd.Int32Dtype`, etc.) is
passed to scikit-learn's `StandardScaler` or `MinMaxScaler`, the conversion to NumPy arrays can
silently produce an **object-dtype array** containing `pd.NA` sentinel objects instead of a numeric
array. Subsequent NumPy reduction operations (`_mean`, `_var`, `_amax`, `_amin`) do not recognise
`dtype=object` as an integer-like type and therefore skip the mandatory integer→float64 promotion
step, causing `pd.NA` values to propagate through arithmetic and produce incorrect results or
`TypeError` exceptions.

**File**: `sklearn/preprocessing/_data.py`

`FLOAT_DTYPES = (np.float64, np.float32, np.float16)`.  That call is a thin wrapper around
`check_array`.
