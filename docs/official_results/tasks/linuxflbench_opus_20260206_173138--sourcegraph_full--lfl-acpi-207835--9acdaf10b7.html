**Buggy Data Structure**: `video_detect_dmi_table` (the DMI quirks table)

**Root Cause**: 
The `video_detect_dmi_table` array is missing a DMI quirk entry for the Acer TravelMate 5735Z laptop. Without this quirk:
1. The kernel detects ACPI video backlight capability
2. The autodetection logic chooses `acpi_backlight_video` (generic ACPI)
3. However, on this laptop the ACPI video backlight interface is broken - it shows the brightness OSD but doesn't actually change the display brightness
4. The native GPU driver (`intel_backlight`) works correctly but isn't selected by default

**Fix**: Add a DMI quirk entry that calls `video_detect_force_native()` for this laptop model, forcing the kernel to use native backlight control. The newer Linux kernel versions include this exact fix at lines 703-711 of `drivers/acpi/video_detect.c`, with explicit reference to bugzilla 207835.
