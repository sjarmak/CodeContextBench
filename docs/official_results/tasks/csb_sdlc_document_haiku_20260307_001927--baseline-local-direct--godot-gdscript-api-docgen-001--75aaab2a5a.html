Frame Layout on Function Call\n\nWhen a GDScript function is called, the VM:\n\n1. **Allocates Stack Frame**:\n   ```\n   alloca_size = sizeof(Variant*) * 3              // Self, class, nil\n              + sizeof(Variant*) * _instruction_args_size  // Instr args\n              + sizeof(Variant) * _stack_size      // Local variables\n   ```\n\n2. **Initializes Fixed Addresses**:\n   ```c\n   stack[ADDR_STACK_SELF] = instance->owner    // \"self\"\n   stack[ADDR_STACK_CLASS] = script              // Class reference\n   stack[ADDR_STACK_NIL] = Variant()             // nil literal\n   ```\n\n3. **Places Arguments on Stack**:\n   ```c\n   for (int i = 0; i < p_argcount; i++) {\n       stack[i + 3] = arguments[i];  // Skip fixed addresses\n   }\n   ```\n\n4. **Initializes Local Variables**:\n   ```c\n   for (int i = p_argcount + 3; i < _stack_size; i++) {\n       new (&stack[i]) Variant();  // Call Variant constructor\n   }\n   ```\n\n5. **Initializes Temporaries** (if any):\n   ```c\n   for (const KeyValue<int, Variant::Type> &E : temporary_slots) {\n       type_init_function_table[E.value](&stack[E.key]);\n   }\n   ```\n\n### Parameter & Return Value Handling\n\n**Parameters**:\n- Stored at stack indices `[3, 3 + argument_count)`\n- Type-checked on entry if function declares types\n- Implicit conversions applied if needed\n\n**Default Arguments**:\n- If `argument_count > provided_argcount`, VM jumps to `OPCODE_JUMP_TO_DEF_ARGUMENT`\n- Code at that location initializes remaining parameters\n\n**Return Value**:\n- Stored in `Variant Frame Layout on Function Call\n\nWhen a GDScript function is called, the VM:\n\n1. **Allocates Stack Frame**:\n   ```\n   alloca_size = sizeof(Variant*) * 3              // Self, class, nil\n              + sizeof(Variant*) * _instruction_args_size  // Instr args\n              + sizeof(Variant) * _stack_size      // Local variables\n   ```\n\n2. **Initializes Fixed Addresses**:\n   ```c\n   stack[ADDR_STACK_SELF] = instance->owner    // \"self\"\n   stack[ADDR_STACK_CLASS] = script              // Class reference\n   stack[ADDR_STACK_NIL] = Variant()             // nil literal\n   ```\n\n3. **Places Arguments on Stack**:\n   ```c\n   for (int i = 0; i < p_argcount; i++) {\n       stack[i + 3] = arguments[i];  // Skip fixed addresses\n   }\n   ```\n\n4. **Initializes Local Variables**:\n   ```c\n   for (int i = p_argcount + 3; i < _stack_size; i++) {\n       new (&stack[i]) Variant();  // Call Variant constructor\n   }\n   ```\n\n5. **Initializes Temporaries** (if any):\n   ```c\n   for (const KeyValue<int, Variant::Type> &E : temporary_slots) {\n       type_init_function_table[E.value](&stack[E.key]);\n   }\n   ```\n\n### Parameter & Return Value Handling\n\n**Parameters**:\n- Stored at stack indices `[3, 3 + argument_count)`\n- Type-checked on entry if function declares types\n- Implicit conversions applied if needed\n\n**Default Arguments**:\n- If `argument_count > provided_argcount`, VM jumps to `OPCODE_JUMP_TO_DEF_ARGUMENT`\n- Code at that location initializes remaining parameters\n\n**Return Value**:\n- Stored in `Variant retvalue`\n- Returned by `OPCODE_RETURN*` instructions\n- Type-coerced if return type declared\n\n### Error Handling\n\n**Call Errors**:\n- `CALL_ERROR_TOO_FEW_ARGUMENTS` - Missing required parameters\n- `CALL_ERROR_TOO_MANY_ARGUMENTS` - Extra arguments passed\n- `CALL_ERROR_INVALID_ARGUMENT` - Type mismatch on parameter\n- `CALL_ERROR_INVALID_METHOD` - Method not found\n\n**Recursion Protection**:\n- `call_depth` counter (thread-local, max 2048)\n- Prevents stack overflow from infinite recursion\n\n### Instruction Arguments\n\nSome instructions (CALL, CONSTRUCT, etc.) use a special instruction arguments format:\n\n```\ncode[ip+1]: arg_count (number of arguments)\ncode[ip+2]: arg[0] address\ncode[ip+3]: arg[1] address\n...\n\nMacro: LOAD_INSTRUCTION_ARGS\n  - Reads arg_count from code[ip+1]\n  - Populates instruction_args[] pointers\n  - Advances ip by 1 to skip arg_count field\n```\n\n**Usage in VM**:\n```c\n#define LOAD_INSTRUCTION_ARGS           \\\n    int instr_arg_count = _code_ptr[ip + 1];    \\\n    for (int i = 0; i < instr_arg_count; i++) { \\\n        GET_VARIANT_PTR(v, i + 1);              \\\n        instruction_args[i] = v;                \\\n    }                                           \\\n    ip += 1;\n```\n\n---\n\n## Variable Access Patterns\n\n### Local Variables\n\n**Declaration**: Variables declared in function scope.\n\n**Bytecode Pattern**:\n```gdscript\nvar x: int = 10\n```\n\nCompiled to:\n```\nOPCODE_ASSIGN_TYPED_BUILTIN dst=[stack_index] src=[const_10] type=INT\n```\n\n**Memory**: Stored in stack frame at calculated offset.\n\n### Parameters\n\n**Declaration**: Function parameters.\n\n**Storage**: Stack slots `[3, 3 + param_count)`\n\n**Bytecode Pattern**:\n```gdscript\nfunc foo(a: int, b: Vector3):\n    pass\n```\n\nWhen called:\n- `stack[3]` = first argument (a)\n- `stack[4]` = second argument (b)\n- Arguments type-checked on entry\n\n### Member Variables\n\n**Declaration**: Instance member fields.\n\n**Access Pattern**:\n```gdscript\nself.member = value  # Write\nx = self.member       # Read\n```\n\n**Bytecode**:\n- Read: `OPCODE_GET_MEMBER dst=[stack] member_index=[field_index]`\n- Write: `OPCODE_SET_MEMBER member_index=[field_index] src=[stack]`\n\n**Memory**: Stored in `GDScriptInstance::members` array (Vector<Variant>)\n\n### Static Variables\n\n**Declaration**: Class-level static variables.\n\n**Access Pattern**:\n```gdscript\nstatic my_var = 10\nmy_var = 20\n```\n\n**Bytecode**:\n- Read: `OPCODE_GET_STATIC_VARIABLE dst=[stack] static_index=[var_index]`\n- Write: `OPCODE_SET_STATIC_VARIABLE static_index=[var_index] src=[stack]`\n\n**Memory**: Stored in `GDScript::static_variables` (Vector<Variant>)\n\n### Global Variables\n\n**Declaration**: Project globals (autoload singletons, constants).\n\n**Access Pattern**:\n```gdscript\nGLOBAL_VAR = value\nx = GLOBAL_VAR\n```\n\n**Bytecode**:\n- Write: `OPCODE_STORE_GLOBAL value=[stack] global_index=[index]`\n- Read: Compiled as constant folding or looked up at compile time\n\n### Temporary Variables\n\n**Purpose**: Hold intermediate values during ## VM Architecture Overview

The GDScript VM is a **stack-based bytecode interpreter** with the following characteristics:

### Execution Model

- **Dispatch Method**: Uses computed goto (on GCC/Clang) for opcode dispatch, falling back to switch statements on other compilers
- **Instruction Pointer (IP)**: Tracks the current position in the bytecode stream (in integers, not bytes)
- **Stack Frame**: Each function call allocates a stack frame containing:
  - Local variables (Variant types)
  - Temporary values
  - Instruction arguments
  - Fixed addresses (self, class, nil)

### Stack Layout

```
[Stack Frame]
├── Fixed Addresses (3 slots)
│   ├── ADDR_STACK_SELF (0): Object instance (self)
│   ├── ADDR_STACK_CLASS (1): GDScript class reference
│   └── ADDR_STACK_NIL (2): Nil value
├── Arguments (parameter count)
├── Local Variables
└── Temporary Values

[Instruction Arguments Area]
├── Array of Variant* pointers for current instruction
└── Used for multi-operand operations
```

### Key VM Properties

- **Max Call Depth**: 2,048 (prevents stack overflow from infinite recursion)
- **Address Bits**: 24 bits for address value + 2 bits for address type = 26 bits per address
