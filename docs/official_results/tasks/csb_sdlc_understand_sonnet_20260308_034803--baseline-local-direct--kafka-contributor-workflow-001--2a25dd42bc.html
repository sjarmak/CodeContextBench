**What the guide covers:**

1. **Build Prerequisites** — Java 8/11/17/21 (21 recommended), Scala 2.13 default (2.12 deprecated), Gradle wrapper bundled in the repo, optional Docker and Python/ducktape for system tests.

2. **Gradle Build System** — Uses `./gradlew` (wrapper, no install needed). The project has ~30+ modules declared in `settings.gradle` (clients, core, streams, connect, raft, metadata, storage, etc.). Key commands: `./gradlew jar`, `./gradlew tasks`, `./gradlew core:jar`, `./gradlewAll jar` (multi-Scala).

3. **Running Tests** — Three tiers:
   - **Unit/integration**: `./gradlew test`, `./gradlew unitTest`, `./gradlew integrationTest`, with per-module (`clients:test`) and per-class/method targeting (`--tests ClassName.methodName`).
   - **System tests**: ducktape-based, run via `bash tests/docker/run_tests.sh` after `./gradlew systemTestLibs`.

4. **CI Pipeline** — **Jenkins** (`Jenkinsfile`) is primary: 4 parallel stages (JDK 8+Scala 2.12, JDK 11, 17, 21 with Scala 2.13), each with validation (Checkstyle, SpotBugs, Spotless) + tests, 8-hour timeout. **GitHub Actions** (`.github/workflows/`) handles Docker image workflows and stale PR management (90-day stale label, 30-day close window).

5. **Code Review Process** — JIRA tickets at issues.apache.org/jira/browse/KAFKA, no enforced branch naming, PRs go to `trunk`, committer must approve and merge, Spotless import formatting required, tests expected for all behavior changes.

6. **Workflow Example** — Full step-by-step from fork/clone → branch → code → style checks → tests → PR → merge.

# Kafka Contributor Guide

## 1. Build Prerequisites

### Java
Kafka requires Java to build and run. Supported versions are **Java 8, 11, 17, and 21**.

- Java 8 and Java 11 support are **deprecated** and targeted for removal in Kafka 4.0.
- The build sets `-release 8` in javac/scalac so binaries are compatible with Java 8+ regardless of which JDK you compile with.
- For `spotlessApply` and `spotlessCheck` (import ordering), use **JDK 11 or 17** — there is a known issue with Java 21.

### Scala
Kafka supports Scala **2.12** and **2.13**. Scala 2.13 is the default. Scala 2.12 is deprecated and will be removed in Kafka 4.0.

The Scala version used for compilation is controlled by the `-PscalaVersion` Gradle flag.

### Other Tools
- **Git** — for source control.
- **Gradle Wrapper** (`./gradlew`) — bundled in the repo; no separate Gradle installation needed.
- **Maven** — only needed for the Kafka Streams quickstart archetype (`streams/quickstart`).
- **Docker** — optional, needed to run system tests locally via `tests/docker/run_tests.sh`.
- **Python 3 + ducktape** — needed for system/integration test infrastructure (see `tests/README.md`).
