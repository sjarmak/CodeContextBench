**Severity:** High
**Ticket:** FORMS-2847

## Summary of Changes

### Problem
Form field widgets that specify custom HTML attributes through `build_attrs()` were not properly escaping special characters (quotes, ampersands, angle brackets) in attribute values, causing malformed HTML.

### Root Cause
The `attrs.html` template uses the `stringformat` filter decorated with `@register.filter(is_safe=True)`, which marks the output as safe and prevents Django's template auto-escaping from being applied to attribute values.

### Solution
Modified `/workspace/django/forms/widgets.py` to properly escape attribute values before passing them to templates:

**1. Updated Import (line 18):**
```python
from django.utils.html import conditional_escape, format_html, html_safe
```

**2. Enhanced `Widget.get_context()` method (lines 316-333):**
- Calls `build_attrs()` to merge widget and render-time attributes
- Applies `conditional_escape()` to each attribute value
- Preserves boolean and None values (needed for template logic)
- Respects safe strings (won't double-escape)

### Key Implementation Details
- **Quotes** → escaped as `&quot;`
- **Ampersands** → escaped as `&amp;`
- **Angle brackets** → escaped as `&lt;` and `&gt;`
- **Safe strings** → not double-escaped (respects `mark_safe()`)
- **Boolean values** → preserved for template boolean attribute rendering

# Django Forms Widget Attribute Escaping Fix

## Problem
Widget attributes with special characters (quotes, ampersands, angle brackets) were not properly escaped in HTML output, causing malformed DOM structure.

## Root Cause
The `attrs.html` template uses the `stringformat` filter which is marked as `is_safe=True`, preventing Django's template auto-escaping from being applied to attribute values. This caused unescaped special characters to appear in HTML.

## Solution
Modified `Widget.get_context()` in `/workspace/django/forms/widgets.py` to escape attribute values before passing them to the template using `conditional_escape()`.

### Changes Made
1. Added `conditional_escape` to imports from `django.utils.html`
2. Modified `get_context()` to escape all attribute values (except booleans and None) using `conditional_escape()`

### Key Implementation Details
- Booleans and None values are preserved as-is so template can handle them
- `conditional_escape()` respects safe strings (won't double-escape)
- String values are properly escaped before being passed to template

### Test Coverage
Added three test cases to `test_textinput.py`:
- `test_custom_attrs_with_quotes` - verifies quotes are escaped as `&quot;`
- `test_custom_attrs_with_ampersand` - verifies ampersands are escaped as `&amp;`
- `test_custom_attrs_with_angle_brackets` - verifies angle brackets are escaped as `&lt;` and `&gt;`
