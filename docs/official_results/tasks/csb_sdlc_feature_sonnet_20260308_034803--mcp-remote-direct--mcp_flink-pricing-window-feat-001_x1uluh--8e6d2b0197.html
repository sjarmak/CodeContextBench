# big-code-flink-feat-001: Implement PricingSessionWindow for Financial Trading

## Task

Implement a custom `PricingSessionWindow` assigner in Apache Flink that groups trading events by market session boundaries (e.g., NYSE 09:30-16:00 ET, LSE 08:00-16:30 GMT) rather than fixed time intervals. This is a common requirement in capital markets streaming analytics where aggregations must align with trading sessions.

The implementation must follow Flink's existing windowing architecture:

1. **PricingSessionWindow** (`flink-streaming-java/src/main/java/org/apache/flink/streaming/api/windowing/assigners/PricingSessionWindow.java`):
   - Extends `MergingWindowAssigner<Object, TimeWindow>`
   - `assignWindows()`: Given an element's timestamp, determines which trading session it belongs to and returns a `TimeWindow(sessionOpen, sessionClose)` for that session
   - `mergeWindows()`: Delegates to `TimeWindow.mergeWindows()` for overlapping window consolidation
   - Factory method: `PricingSessionWindow.forMarket(String marketId, ZoneId timezone, LocalTime open, LocalTime close)`
   - Must handle pre/post-market sessions and overnight sessions (e.g., futures markets)
   - Returns `EventTimeTrigger` as default trigger

2. **PricingSessionTrigger** (`flink-streaming-java/src/main/java/org/apache/flink/streaming/api/windowing/triggers/PricingSessionTrigger.java`):
   - Extends `Trigger<Object, TimeWindow>`
   - Fires at market close (window end) via event-time timer
   - Supports early firing on configurable events (e.g., circuit breaker halts)
   - `canMerge()` returns `true`, `onMerge()` re-registers timers
   - `clear()` cleans up all registered timers

3. **TradingSessionExtractor** (`flink-streaming-java/src/main/java/org/apache/flink/streaming/api/windowing/assigners/TradingSessionExtractor.java`):
   - Functional interface for extracting market ID from stream elements
   - Modeled after `SessionWindowTimeGapExtractor`
   - Enables dynamic session assignment based on element content

Study existing window implementations, particularly `EventTimeSessionWindows` and `DynamicEventTimeSessionWindows`, for the complete pattern.

## Analysis

### PricingSessionWindow Design

The window assigner converts each element's event-time timestamp into a fixed `TimeWindow` representing a single trading session:

1. **Timezone conversion**: Uses Java's `ZonedDateTime` + `ZoneId` to convert epoch milliseconds to a local time in the market's timezone, correctly handling DST transitions.

2. **Session date determination**:
   - If the event timestamp falls **before or during** the session (local time < sessionClose), it is assigned to the **current day's** session window.
   - If the event timestamp falls **after** the session close (local time >= sessionClose), it is assigned to the **next day's** session window. This handles post-market events correctly.

3. **Window boundaries**: The resulting `TimeWindow(sessionOpenMs, sessionCloseMs)` uses epoch milliseconds for the session open and close on the determined trading day.

4. **Merging**: Since all events within the same trading day produce identical `TimeWindow` instances (same start/end), `TimeWindow.mergeWindows()` handles the standard overlapping/consecutive window consolidation as used by session windows.

### PricingSessionTrigger Design

Modeled directly after `EventTimeTrigger` with additions for capital markets:

- **Normal firing**: Fires when the watermark passes `window.maxTimestamp()` (market close time), via `ctx.registerEventTimeTimer(window.maxTimestamp())`.
- **Early firing**: Supports circuit breaker halts or other extraordinary events via `signalEarlyFire(ctx)` which sets a `ValueState<Boolean>` flag. On the next `onElement()` call, the trigger detects the flag and fires immediately.
- **Merge support**: `canMerge()` returns `true`; `onMerge()` merges the early-fire state partition and re-registers the timer for the merged window's end timestamp.
- **Cleanup**: `clear()` deletes the event-time timer and clears the early-fire state.

### TradingSessionExtractor Design

A `@FunctionalInterface` extending `Serializable`, modeled after `SessionWindowTimeGapExtractor`. It enables dynamic routing of stream elements to different markets without requiring separate keyed streams per market — the extractor can be composed with a dynamic `PricingSessionWindow` variant to support multi-market topologies.

### Integration with Flink Windowing Architecture

All three classes integrate cleanly with Flink's existing windowing framework:
- `PricingSessionWindow` extends `MergingWindowAssigner<Object, TimeWindow>` from `flink-runtime`, consistent with `EventTimeSessionWindows`
- `PricingSessionTrigger` extends `Trigger<Object, TimeWindow>` from `flink-runtime`, consistent with `EventTimeTrigger`
- Both classes are placed in `flink-streaming-java` (higher-level module), not `flink-runtime` (lower-level), matching the placement of `EventTimeSessionWindows` and other concrete window implementations
- Imports only use classes already present in the Flink dependency graph (`java.time.*`, `flink-runtime` windowing classes)
