## Bug Report

**Bug ID**: 207835
**Summary**: ACPI video backlight brightness control does not work on Acer TravelMate 5735Z by default

On the Intel GM45/GMA 4500MHD laptop with Debian Sid/unstable and Linux 5.6.7 and GNOME Shell 3.36.2, trying to change the brightness of the internal display using the function keys, GNOME shows the OSD, which seems to have five levels (from acer_wmi?), but the actual brightness does *not* change. There is only `/sys/class/backlight/intel_backlight/brightness` though, and writing values to it seems to work.

Booting the system with `acpi_backlight=vendor`, exposes `/sys/devices/platform/acer-wmi/backlight/acer-wmi/brightness`, but the behavior is the same.

Booting the system with `acpi_backlight=native` or `acpi_backlight=native`, changing the brightness with the function keys works, and there now seem to be 15 (or 16) levels.

- **Device**: Acer TravelMate 5735Z (Intel GM45/GMA 4500MHD)
- **Issue**: ACPI video backlight brightness control doesn't work by default
- **Kernel**: Linux 5.6.7

## Root Cause Analysis

### The Problem
1. Default boot: Brightness control via GNOME doesn't work
2. `acpi_backlight=vendor`: Exposes acer-wmi backlight but doesn't work
3. `acpi_backlight=native`: Works correctly with 15-16 brightness levels

### Key Evidence
1. Booting with `acpi_backlight=native` makes brightness control work
2. Writing to `/sys/class/backlight/intel_backlight/brightness` works manually
3. Device has Intel GM45 chipset with i915 GPU (supports native backlight)
4. BIOS from 2011 (pre-Windows 8 era)

### Technical Root Cause
The device auto-detection logic in `acpi_video_get_backlight_type()` falls through to the default case:

1. Acer TravelMate 5735Z is NOT in `video_detect_dmi_table`
2. Auto-detection checks if device has ACPI_VIDEO_BACKLIGHT capability (YES)
3. Windows 8 check fails (2011 BIOS, pre-Windows 8)
4. Returns `acpi_backlight_video` (default fallback)
5. But ACPI video backlight doesn't work on this hardware

When acer-wmi module loads:
1. Device NOT in acer-wmi's `video_vendor_dmi_table`
2. Calls `acpi_video_get_backlight_type()` which returns `acpi_backlight_video`
3. Since it's not `acpi_backlight_vendor`, acer-wmi disables its brightness
4. Result: ACPI video becomes the backlight (which doesn't work)

The i915 GPU driver registers a BACKLIGHT_RAW device when native backlight is forced,
