fetched.\n\n```java\nlong offset = consumer.position(partition);\n// If last fetched record was offset 100, position is 101\n```\n\n**Characteristics:**\n\n- Automatically advances when records are fetched via `poll()`\n- Can be manually set with `seek()`\n- Not persisted; resets after consumer restart\n\n#### Committed Offset\n\nThe last offset successfully committed to Kafka.\n\n```java\nMap<TopicPartition, OffsetAndMetadata> committed = consumer.committed(\n    Collections.singleton(partition)\n);\nOffsetAndMetadata oam = committed.get(partition);\nlong offset = oam.offset();  // 0 if never committed\n```\n\n**Characteristics:**\n\n- Persisted in `__consumer_offsets` topic\n- Used to resume after consumer restart\n- Can be committed via `commitSync()` or `commitAsync()`\n- Can be auto-committed based on time interval\n\n#### Offset Semantics\n\n**Critical Rule: Committed offset = next offset to read**\n\n```java\n// WRONG:\nConsumerRecord record = getLastRecord();\nconsumer.commitSync(record.offset());  // Commits record itself\n\n// RIGHT:\nConsumerRecord record = getLastRecord();\nconsumer.commitSync(record.offset() + 1);  // Commits offset of NEXT record\n\n// Or:\nconsumer.commitSync(Collections.singletonMap(partition,\n    new OffsetAndMetadata(record.offset() + 1)));\n```\n\n### Automatic Offset Committing\n\nWhen enabled, offsets are committed periodically:\n\n```properties\nenable.auto.commit=true              # Enable auto-commit\nauto.commit.interval.ms=5000         # Commit every 5 seconds\n```\n\n**Behavior:**\n\n- Commits all offsets from most recent `poll()` call\n- Offset committed = last fetched offset + 1\n- Happens asynchronously before next `poll()` call\n- No exception if commit fails (logged but not raised)\n\n**Risks:**\n\n- Records lost if consumer crashes between fetch and commit\n- At-least-once semantics: records may be re-delivered on restart\n- Cannot distinguish between processed and unprocessed records\n\n**Recommendation:** Disable for exactly-once semantics\n\n### Manual Offset Committing\n\n#### Synchronous Commit: `commitSync()`\n\n```java\ntry {\n    consumer.commitSync();  // Commits all offsets from last poll()\n} catch (CommitFailedException e) {\n    // Consumer no longer part of group, cannot commit\n    logger.error(\"Commit failed\", e);\n}\n```\n\n**Behavior:**\n\n- Blocks until commit succeeds or fails\n- Uses timeout from `default.api.timeout.ms`\n- Commits offsets for all fetched records from last `poll()` call\n- Fails immediately if `max.poll.interval.ms` exceeded (proactive expulsion)\n\n**Common Exception:**\n\n```\nCommitFailedException: The consumer is not part of an active consumer group\n```\n\nCauses:\n- Consumer not yet subscribed/joined group\n- Exceeded `max.poll.interval.ms` without calling `poll()`\n- Another consumer in group claims ownership of partition\n- Consumer was explicitly removed from group\n\n#### Synchronous Commit with Timeout: `commitSync(Duration timeout)`\n\n```java\nconsumer.commitSync(Duration.ofSeconds(5));  // Custom timeout\n```\n\n#### Explicit Offset Commit: `commitSync(Map<TopicPartition, OffsetAndMetadata> offsets)`\n\nCommit specific offsets (e.g., after processing only some records).\n\n```java\nMap<TopicPartition, OffsetAndMetadata> offsets = new HashMap<>();\nfor (TopicPartition tp : records.partitions()) {\n    List<ConsumerRecord<String, String>> partitionRecords = records.records(tp);\n\n    // Find last successfully processed record\n    ConsumerRecord<String, String> lastProcessed = findLastSuccessful(partitionRecords);\n\n    if (lastProcessed != null) {\n        offsets.put(tp, new OffsetAndMetadata(\n            lastProcessed.offset() + 1,\n            Optional.of(lastProcessed.leaderEpoch().orElse(-1)),\n            \"processed\"\n        ));\n    }\n}\n\nconsumer.commitSync(offsets);  // Commit only processed offsets\n```\n\n#### Asynchronous Commit: `commitAsync()`\n\n```java\nconsumer.commitAsync((offsets, exception) -> {\n    if (exception != null) {\n        logger.error(\"Commit failed for offsets: {}\", offsets, exception);\n        // Retry or store for later commit\n    } else {\n        logger.info(\"Commit succeeded\");\n    }\n});\n\n// Continue polling immediately, callback invoked later\nConsumerRecords<String, String> records = consumer.poll(Duration.ofMillis(100));\n```\n\n**Characteristics:**\n\n- Returns immediately without waiting\n- Callback invoked when commit result available\n- Multiple pending async commits: callbacks invoke in order\n- Failures typically retried automatically\n\n#### Asynchronous Commit with Callback: `commitAsync(Map<TopicPartition, OffsetAndMetadata> offsets, OffsetCommitCallback callback)`\n\n```java\nOffsetCommitCallback callback = (offsets, exception) -> {\n    if (exception != null) {\n        logger.error(\"Async commit failed\", exception);\n    }\n};\n\nconsumer.commitAsync(Collections.singletonMap(\n    partition,\n    new OffsetAndMetadata(nextOffset)\n), callback);\n```\n\n### Offset Commit Failures\n\n#### Retryable Failures\n\nThe consumer automatically retries:\n- Temporary network failures\n- Broker temporarily unavailable\n- Broker leadership changes during commit\n\n#### Non-Retryable Failures\n\nThe consumer does NOT retry:\n- `CommitFailedException`: consumer expelled from group\n- `AuthorizationException`: no permission to commit\n- Offset metadata too large (> 1MB)\n\n### Offset Reset Strategies\n\nIf position cannot be determined (no committed offset, consumer reset):\n\n```properties\nauto.offset.reset=earliest    # Start from beginning\nauto.offset.reset=latest      # Start from end\nauto.offset.reset=none        # Raise NoOffsetForPartitionException\n```\n\n**Applicable When:**\n\n- Consumer first joins group (no prior committed offset)\n- Partition added after consumer started\n- Consumer explicitly calls `seekToBeginning()` or `seekToEnd()`\n\n---\n\n## Flow Control and Position Management\n\n### Pause and Resume\n\n#### `pause(Collection<TopicPartition> partitions)`\n\nStop fetching from specified partitions.\n\n```java\n// Pause high-latency partition\nconsumer.pause(Collections.singleton(slowPartition));\n\n// Continue polling for other partitions\nConsumerRecords<String, String> records = consumer.poll(Duration.ofMillis(100));\n// slowPartition not included even if assigned\n```\n\n**Use Cases:**\n\n1. **Backpressure**: Downstream system is slow\n2. **Priority partitions**: Focus on high-priority topics first\n3. **Load balancing**: Redirect processing capacity\n4. **Throttling**: Rate-limit consumption\n\n**Important Notes:**\n\n- Paused partitions still count as \"assigned\"\n- Partition state does NOT persist through rebalance\n- `pause()` called during rebalance callback affects new assignment\n- Multiple `pause()` calls are cumulative\n\n#### `resume(Collection<TopicPartition> partitions)`\n\nResume fetching from paused partitions.\n\n```java\nconsumer.resume(Collections.singleton(slowPartition));\n```\n\n#### `paused()`\n\nGet the set of currently paused partitions.\n\n```java\nSet<TopicPartition> paused = consumer.paused();\n```\n\n### Position Control\n\n#### `seek(TopicPartition partition, long offset)`\n\nSet the offset to fetch on next `poll()`.\n\n```java\n// Seek to specific offset\nconsumer.seek(partition, 12345);\n\n// Next poll() starts from offset 12345\nConsumerRecords<String, String> records = consumer.poll(Duration.ofMillis(100));\n// First record will be offset 12345 (or next available)\n```\n\n**Semantics:**\n\n- Position is set immediately\n- Fetching starts at this offset on next `poll()`\n- Offset must be non-negative\n- Partition must be assigned to consumer\n\n**Exceptions:**\n\n- `IllegalArgumentException`: offset < 0\n- `IllegalStateException`: partition not assigned\n\n#### `seek(TopicPartition partition, OffsetAndMetadata offsetAndMetadata)`\n\nSeek with leader epoch information (for truncation detection).\n\n```java\nOffsetAndMetadata offsetWithEpoch = new OffsetAndMetadata(\n    12345,\n    Optional.of(leaderEpoch),  // Include leader epoch\n    \"\"\n);\nconsumer.seek(partition, offsetWithEpoch);\n```\n\n#### `position(TopicPartition partition)`\n\nGet the current position (next offset to fetch).\n\n```java\nlong nextOffset = consumer.position(partition);\n```\n\n**Behavior:**\n\n- Performs remote call if position not yet initialized\n- Blocks up to `default.api.timeout.ms`\n- Returns offset of next record to be fetched\n\n#### `position(TopicPartition partition, Duration timeout)`\n\nPosition with custom timeout.\n\n```java\nlong pos = consumer.position(partition, Duration.ofSeconds(10));\n```\n\n### Seeking to Beginning and End\n\n#### `seekToBeginning(Collection<TopicPartition> partitions)`\n\nSeek to the first available offset.\n\n```java\nconsumer.seekToBeginning(Collections.singleton(partition));\n// or all assigned partitions\nconsumer.seekToBeginning(Collections.emptyList());\n```\n\n**Behavior:**\n\n- Lazy evaluation: actual seek happens at next `poll()` call\n- Sets position to earliest offset in partition\n- Use `beginningOffsets()` to query without seeking\n\n#### `seekToEnd(Collection<TopicPartition> partitions)`\n\nSeek to the last offset.\n\n```java\nconsumer.seekToEnd(Collections.singleton(partition));\n```\n\n**Behavior:**\n\n- Lazy evaluation: actual seek happens at next `poll()` call\n- Sets position just past last message\n- If `isolation.level=read_committed`: seeks to Last Stable Offset (LSO)\n- Next `poll()` returns no records for this partition (unless new messages arrive)\n\n### Timestamp-Based Seeking\n\n#### `offsetsForTimes(Map<TopicPartition, Long> timestampsToSearch)`\n\nFind offsets for specific timestamps.\n\n```java\n// Find offset for records >= 1 hour ago\nlong oneHourAgo = System.currentTimeMillis() - 3600_000;\nMap<TopicPartition, Long> timestampMap = Collections.singletonMap(\n    partition,\n    oneHourAgo\n);\n\nMap<TopicPartition, OffsetAndTimestamp> result = consumer.offsetsForTimes(timestampMap);\n\nOffsetAndTimestamp oat = result.get(partition);\nif (oat != null) {\n    consumer.seek(partition, oat.offset());\n}\n```\n\n**Behavior:**\n\n- Returns earliest offset whose timestamp >= requested timestamp\n- Returns `null` if no matching offset found\n- Requires topic configured with `message.format.version >= 0.10`\n- Blocking call up to `default.api.timeout.ms`\n\n**Timestamps:**\n\n- Created by producer: `CreateTime` (default)\n- Added by broker: `LogAppendTime` (if configured)\n- Broker can have log compaction without timestamps\n\n### Beginning and End Offset Queries\n\n#### `beginningOffsets(Collection<TopicPartition> partitions)`\n\nQuery first available offset (doesn't seek).\n\n```java\nMap<TopicPartition, Long> beginnings = consumer.beginningOffsets(\n    Collections.singleton(partition)\n);\nlong firstOffset = beginnings.get(partition);\n```\n\n#### `endOffsets(Collection<TopicPartition> partitions)`\n\nQuery offset after last message (doesn't seek).\n\n```java\nMap<TopicPartition, Long> ends = consumer.endOffsets(\n    Collections.singleton(partition)\n);\nlong nextOffset = ends.get(partition);  // Offset to append next message\n```\n\n**Difference from seekToEnd:**\n- These methods only query; don't change position\n- Can be called even if partition not assigned\n- Useful for monitoring lag or validating offsets\n\n### Current Lag Calculation\n\n#### `currentLag(TopicPartition topicPartition)`\n\nGet approximate consumer lag.\n\n```java\nOptionalLong lag = consumer.currentLag(partition);\nif (lag.isPresent()) {\n    System.out.println(\"Lag: \" + lag.getAsLong() + \" records\");\n}\n```\n\n**Behavior:**\n\n- Returns `OptionalLong.empty()` if lag cannot be determined\n- Lag = `endOffset - position`\n- Based on cached metadata; may be approximate\n\n---\n\n## Rebalancing and Consumer Groups\n\n### Rebalance Triggers\n\nA rebalance occurs when:\n\n1. **New consumer joins group** (first `poll()` after subscribe)\n2. **Consumer leaves group** (`close()`, `unsubscribe()`, session timeout)\n3. **Partition count changes** (in subscribed topics)\n4. **Topic added** (if using regex subscription)\n5. **Consumer explicitly requests** (`enforceRebalance()`)\n\n### Rebalance Protocol\n\n**Classic (Eager) Rebalancing:**\n\n1. Stop: All consumers stop fetching\n2. Revoke: All consumers lose all partitions (even unchanged ones)\n3. Rebalance: Broker runs assignment strategy\n4. Assign: Consumers receive new assignment\n5. Resume: Consumers begin fetching\n\n**Cooperative Rebalancing:**\n\n1. Revoke: Consumers revoke only partitions they're losing\n2. Rebalance: Broker runs assignment strategy\n3. Assign: Consumers receive only new partitions\n4. Resume: Seamless transition with minimal stop\n\n### Consumer Rebalance Listener\n\n#### Implementing the Interface\n\n```java\npublic class MyRebalanceListener implements ConsumerRebalanceListener {\n    private final KafkaConsumer<?, ?> consumer;\n    private final Database database;\n\n    public MyRebalanceListener(KafkaConsumer<?, ?> consumer, Database database) {\n        this.consumer = consumer;\n        this.database = database;\n    }\n\n    @Override\n    public void onPartitionsRevoked(Collection<TopicPartition> partitions) {\n        // CRITICAL: Commit offsets before losing partitions\n        try {\n            consumer.commitSync();\n        } catch (CommitFailedException e) {\n            logger.error(\"Failed to commit\", e);\n        }\n\n        // Clean up partition-local state\n        for (TopicPartition tp : partitions) {\n            flushPartitionCache(tp);\n        }\n    }\n\n    @Override\n    public void onPartitionsAssigned(Collection<TopicPartition> partitions) {\n        // Initialize state for new partitions\n        for (TopicPartition tp : partitions) {\n            long offset = database.getOffset(tp);\n            if (offset >= 0) {\n                consumer.seek(tp, offset);\n            }\n        }\n    }\n\n    @Override\n    public void onPartitionsLost(Collection<TopicPartition> partitions) {\n        // Partitions already reassigned; don't commit\n        logger.warn(\"Partitions lost without revoke callback: {}\", partitions);\n    }\n}\n\n// Register listener\nconsumer.subscribe(Arrays.asList(\"topic1\", \"topic2\"),\n    new MyRebalanceListener(consumer, database));\n```\n\n#### Callback Execution Context\n\n**When Called:**\n\n- Only within `poll()` calls\n- Before other operations resume\n- Synchronously (callback must return before `poll()` returns)\n\n**Exception Handling:**\n\n- Exceptions from callbacks propagate to the `poll()` call\n- Consumer is still notified of assignment (callback failure doesn't prevent assignment)\n- `WakeupException` can be raised from nested consumer calls\n\n**Callback Constraints:**\n\n- Cannot call `poll()` recursively\n- Can call `commitSync()`, `seek()`, `position()`, etc.\n- Cannot call `subscribe()`, `assign()`, `unsubscribe()`, `close()`\n\n### Proactive Consumer Expulsion\n\nIf consumer doesn't call `poll()` within `max.poll.interval.ms`:\n\n1. Consumer is expelled from group\n2. Partitions reassigned to other members\n3. Next `commitSync()` call raises `CommitFailedException`\n\n```properties\nmax.poll.interval.ms=300000  # 5 minutes default\n```\n\n**Recovery:**\n\n```java\ntry {\n    consumer.commitSync();\n} catch (CommitFailedException e) {\n    // Rejoin group by calling poll()\n    consumer.poll(Duration.ZERO);\n    // Then retry commit\n    consumer.commitSync();\n}\n```\n\n### Enforcing Rebalance\n\n#### `enforceRebalance()` and `enforceRebalance(String reason)`\n\nTrigger rebalance on next `poll()` call.\n\n```java\nconsumer.enforceRebalance(\"Rebalancing due to external event\");\n\n// Rebalance happens on next poll()\nconsumer.poll(Duration.ofMillis(100));  // Triggers rebalance\n```\n\n**Use Cases:**\n\n- External coordination signals need rebalancing\n- Graceful shutdown preparation\n- Handling partition assignment changes\n\n---\n\n## Metadata and Monitoring\n\n### Consumer Metadata\n\n#### `groupMetadata()`\n\nGet current group membership information.\n\n```java\nConsumerGroupMetadata metadata = consumer.groupMetadata();\n\nString groupId = metadata.groupId();        // Consumer group ID\nint generation = metadata.generationId();   // Generation ID (increments on rebalance)\nString memberId = metadata.memberId();      // Member ID assigned by broker\nOptional<String> groupInstanceId = metadata.groupInstanceId();  // Static assignment ID\n```\n\n**Use Cases:**\n\n- Pass to producer for transactional writes\n- Track generation for rebalance detection\n- Identify static group members\n\n#### `clientInstanceId(Duration timeout)`\n\nGet unique instance ID.\n\n```java\nUuid instanceId = consumer.clientInstanceId(Duration.ofSeconds(5));\n```\n\n### Partition Information\n\n#### `partitionsFor(String topic)` and `partitionsFor(String topic, Duration timeout)`\n\nList partitions for a topic.\n\n```java\nList<PartitionInfo> partitions = consumer.partitionsFor(\"topic1\");\n\nfor (PartitionInfo info : partitions) {\n    System.out.println(\"Partition: \" + info.partition());\n    System.out.println(\"  Leader: \" + info.leader());\n    System.out.println(\"  Replicas: \" + Arrays.toString(info.replicas()));\n    System.out.println(\"  In-Sync Replicas: \" + Arrays.toString(info.inSyncReplicas()));\n}\n```\n\n**Behavior:**\n\n- Performs remote call if topic metadata not cached\n- Blocks up to timeout\n- Returns null for unknown topics\n\n#### `listTopics()` and `listTopics(Duration timeout)`\n\nList all topics in cluster.\n\n```java\nMap<String, List<PartitionInfo>> allTopics = consumer.listTopics();\n\nfor (Map.Entry<String, List<PartitionInfo>> entry : allTopics.entrySet()) {\n    System.out.println(\"Topic: \" + entry.getKey());\n    System.out.println(\"  Partitions: \" + entry.getValue().size());\n}\n```\n\n### Metrics\n\n#### `metrics()`\n\nGet consumer metrics (network I/O, latencies, etc.).\n\n```java\nMap<MetricName, ? extends Metric> metrics = consumer.metrics();\n\nfor (Map.Entry<MetricName, ? extends Metric> entry : metrics.entrySet()) {\n    MetricName name = entry.getKey();\n    Metric metric = entry.getValue();\n\n    System.out.println(name.name() + \": \" + metric.value());\n}\n```\n\n**Common Metrics:**\n\n- `fetch-latency-avg`, `fetch-latency-max`: Fetch request latency\n- `records-consumed-total`: Total records fetched\n- `bytes-consumed-total`: Total bytes fetched\n- `commit-latency-avg`: Offset commit latency\n\n---\n\n## Exception Handling\n\n### Consumer Exceptions\n\n#### CommitFailedException\n\n```java\ntry {\n    consumer.commitSync();\n} catch (CommitFailedException e) {\n    // Consumer no longer owns partitions\n    logger.error(\"Cannot commit: not in group or partitions revoked\");\n}\n```\n\n**Causes:**\n\n- Consumer expelled from group (max.poll.interval.ms exceeded)\n- Partition reassigned to another consumer\n- Consumer not subscribed to any topics\n- Committing to unassigned partition\n\n**Recovery:**\n\n```java\ntry {\n    consumer.commitSync();\n} catch (CommitFailedException e) {\n    // Rejoin group\n    consumer.poll(Duration.ZERO);\n\n    // Retry commit\n    try {\n        consumer.commitSync();\n    } catch (CommitFailedException e2) {\n        logger.error(\"Still cannot commit after rejoin\");\n    }\n}\n```\n\n#### RebalanceInProgressException\n\n```java\ntry {\n    consumer.commitSync();\n} catch (RebalanceInProgressException e) {\n    // Rebalance in progress, try poll() first\n    consumer.poll(Duration.ZERO);\n    consumer.commitSync();  // Retry\n}\n```\n\n**When Raised:**\n\n- Calling `commitSync()` during rebalance (though callbacks usually prevent this)\n- Rebalance started but not yet complete\n\n#### WakeupException\n\n```java\ntry {\n    while (true) {\n        consumer.poll(Duration.ofMillis(100));\n    }\n} catch (WakeupException e) {\n    logger.info(\"Consumer awakened, shutting down\");\n    consumer.close();\n}\n```\n\n**Characteristics:**\n\n- Non-fatal: signals consumer should stop\n- Raised in `poll()` or other blocking calls\n- Thread-safe `wakeup()` method from another thread triggers it\n- Does NOT close consumer automatically\n\n#### InterruptException\n\n```java\ntry {\n    consumer.poll(Duration.ofMillis(100));\n} catch (InterruptException e) {\n    Thread.currentThread().interrupt();\n    logger.info(\"Consumer thread interrupted\");\n}\n```\n\n**When Raised:**\n\n- Consumer thread interrupted while blocked in API call\n- Similar to `WakeupException` but from thread interruption\n\n#### OffsetOutOfRangeException\n\n```java\ntry {\n    consumer.poll(Duration.ofMillis(100));\n} catch (OffsetOutOfRangeException e) {\n    logger.error(\"Offset out of range, seeking to beginning\");\n    consumer.seekToBeginning(e.partitions());\n    consumer.poll(Duration.ZERO);\n}\n```\n\n**When Raised:**\n\n- Position is beyond end of partition\n- Position is before beginning (deleted due to retention)\n- Offset reset strategy is `none`\n\n#### NoOffsetForPartitionException\n\n```java\ntry {\n    consumer.poll(Duration.ofMillis(100));\n} catch (NoOffsetForPartitionException e) {\n    logger.error(\"No offset for partition, using default strategy\");\n    // auto.offset.reset applies\n    consumer.poll(Duration.ZERO);\n}\n```\n\n**When Raised:**\n\n- No committed offset for partition\n- Partition new to consumer\n- Offset reset strategy is `none`\n\n#### InvalidOffsetException\n\n```java\ntry {\n    long pos = consumer.position(partition);\n} catch (InvalidOffsetException e) {\n    logger.error(\"Invalid position, resetting\");\n    consumer.seekToBeginning(Collections.singleton(partition));\n}\n```\n\n**When Raised:**\n\n- Current position cannot be determined\n- Requested offset is invalid\n- Log compaction has removed all committed offset data\n\n#### TimeoutException\n\n```java\ntry {\n    consumer.commitSync(Duration.ofSeconds(5));\n} catch (TimeoutException e) {\n    logger.error(\"Commit timeout, will retry asynchronously\");\n    consumer.commitAsync();\n}\n```\n\n**When Raised:**\n\n- API call timeout exceeded\n- Broker unavailable for entire duration\n- No response received from broker\n\n#### AuthenticationException and AuthorizationException\n\n```java\ntry {\n    consumer.poll(Duration.ofMillis(100));\n} catch (AuthenticationException e) {\n    logger.error(\"Authentication failed: {}\", e.getMessage());\n    // Check credentials and restart\n} catch (AuthorizationException e) {\n    logger.error(\"Not authorized to access topic: {}\", e.getMessage());\n    // Check ACLs\n}\n```\n\n#### FencedInstanceIdException\n\n```java\ntry {\n    consumer.poll(Duration.ofMillis(100));\n} catch (FencedInstanceIdException e) {\n    logger.error(\"Consumer instance fenced by broker\");\n    consumer.close();\n    // Create new consumer with same group.instance.id to rejoin\n}\n```\n\n**When Raised:**\n\n- Using static group membership with `group.instance.id`\n- Another instance with same ID joined group\n- Original instance should shut down\n\n### Exception Recovery Patterns\n\n#### Graceful Degradation\n\n```java\nlong retries = 0;\nlong maxRetries = 3;\n\nwhile (retries < maxRetries) {\n    try {\n        consumer.poll(Duration.ofMillis(100));\n        retries = 0;  // Reset on success\n    } catch (RetriableException e) {\n        retries++;\n        if (retries >= maxRetries) {\n            logger.error(\"Max retries exceeded\");\n            throw e;\n        }\n        Thread.sleep(1000 * retries);  // Exponential backoff\n    }\n}\n```\n\n#### Partition-Level Recovery\n\n```java\ntry {\n    ConsumerRecords<String, String> records = consumer.poll(Duration.ofMillis(100));\n\n    for (TopicPartition tp : records.partitions()) {\n        try {\n            processPartition(records.records(tp));\n            consumer.commitSync(Collections.singletonMap(tp,\n                new OffsetAndMetadata(getNextOffset(tp))));\n        } catch (Exception e) {\n            logger.error(\"Error processing partition {}\", tp, e);\n            consumer.seek(tp, consumer.position(tp));  // Retry from same offset\n        }\n    }\n} catch (WakeupException | InterruptException e) {\n    // Graceful shutdown\n    consumer.close();\n}\n```\n\n---\n\n## Configuration Properties\n\n### Critical Configuration\n\n| Property | Default | Impact ## Overview and Architecture

### Purpose

The `KafkaConsumer` is the primary client for consuming records from Apache Kafka topics. It provides:

- **Scalability**: Multiple consumer instances in a group distribute partitions for parallel processing
- **Fault tolerance**: Automatic failure detection with rebalancing and group coordination
- **Flexibility**: Support for both automatic (group-managed) and manual partition assignment
- **Offset management**: Track consumed positions with configurable commit strategies
- **Flow control**: Pause/resume consumption at the partition level

### Key Characteristics

- **Single-threaded design**: A `KafkaConsumer` instance is not thread-safe except for the `wakeup()` method
