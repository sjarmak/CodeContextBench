Full-page caching middleware for Django.

This module provides full-page caching based on request URL and configured cache
backends. When enabled, Django caches entire responses that match specific criteria,
allowing subsequent requests for the same URL to be served from cache without
hitting the view layer.

Configuration
=============

The caching behavior is controlled by Django settings:

- ``CACHE_MIDDLEWARE_SECONDS`` (int): Default cache timeout in seconds for all
  cacheable responses. Used when the response's Cache-Control header does not
  specify max-age. Defaults to 600 seconds.

- ``CACHE_MIDDLEWARE_KEY_PREFIX`` (str): A prefix to add to all cache keys.
  Useful for distinguishing cache keys when multiple Django instances share
  the same cache backend. Defaults to empty string.

- ``CACHE_MIDDLEWARE_ALIAS`` (str): The name of the cache backend to use from
  the CACHES setting. Defaults to 'default'.

Middleware Setup
================

The canonical way to enable cache middleware is to use the two-part middleware
configuration:

1. ``UpdateCacheMiddleware`` as the first middleware (executes last during response)
