**Ticket:** #36138
**Date:** February 7, 2026
**Scope:** Migration from `[(name, email), ...]` tuple format to `[email, ...]` string list format

---

## Summary

Django's `ADMINS` and `MANAGERS` settings are currently implemented as lists of 2-tuples containing (display_name, email_address) pairs. This audit identifies **3 core production files**, **4 middleware/utility files**, **2+ test suites**, and **5 documentation files** that require changes to support the new simple email-string format. The central validation logic in `django/core/mail/__init__.py:135-136` enforces the tuple format via runtime type-checking that must be completely rewritten to accept strings instead.

---

## Root Cause

The tuple format is deeply embedded in Django's email notification system through two core code paths:

1. **Email Dispatch Path** (`django/core/mail/__init__.py`):

   - Both `mail_admins()` and `mail_managers()` wrappers depend on this

   - Class `AdminEmailHandler.emit()` (Line 97) performs conditional check on `settings.ADMINS`
