# Apache Camel Architecture Reference

## High-Level Message Routing Flow

**Model → Runtime Bridge**:
- RouteDefinition (DSL model) → RouteReifier.createRoute() → Route (runtime)
- ProcessorDefinition → ProcessorReifier → Processor + Channel

**Runtime Message Flow**:
1. Consumer receives message from endpoint
2. Consumer calls route.processor.process(exchange)
3. Processor chain (RoutePipeline) executes each processor in sequence
4. Each processor wrapped in Channel (applies interceptors, error handlers)
5. Final processor (Producer via to()) sends message to destination

## Key Classes & Responsibilities

**Hierarchy: Component → Endpoint → Producer/Consumer**
- Component: factory for Endpoints
- Endpoint: factory for Producers/Consumers; creates Exchanges
- Consumer: receives messages, invokes Processor chain
- Producer: extends Processor; sends messages outbound

**Processor Chain Architecture**:
- RoutePipeline: chains multiple processors sequentially
- Channel: wraps processor with interceptors + error handlers
- InternalProcessor: adds cross-cutting concerns (UnitOfWork, policies, advices)

**Reifier Pattern**:
- RouteReifier: converts RouteDefinition → Route
- ProcessorReifier: converts ProcessorDefinition → Processor + Channel
- Routes all outputs recursively through ProcessorReifier.addRoutes()

- `Component.createEndpoint(uri, parameters)` → Endpoint
- `Endpoint.createConsumer(processor)` → Consumer
- `Endpoint.createProducer()` → Producer
- `Processor.process(exchange)` → void (core contract)
