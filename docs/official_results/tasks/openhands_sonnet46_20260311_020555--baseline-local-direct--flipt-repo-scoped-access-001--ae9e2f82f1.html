### Tracking semantics

**`Variant()`**
- `RecordEvaluation()` — always, on entry
- `RecordError()` — flag store failure or evaluation failure
- `RecordMatch()` — when `resp.Match == true`

**`Boolean()`**
- `RecordEvaluation()` — always, on entry
- `RecordError()` — flag store failure, wrong flag type, or evaluation failure
- `RecordMatch()` — when `resp.Reason == MATCH_EVALUATION_REASON`

**`Batch()`**
- `RecordEvaluation()` — once per item, at the top of the loop
- `RecordError()` — flag not found, store error, evaluation failure, or unknown flag type
- `RecordMatch()` — per item, on a successful boolean or variant match

### Design decisions
- **`sync/atomic`** over `sync.Mutex` — no lock contention, minimal overhead, correct for simple integer counters
- **Public API boundary** for tracking — avoids double-counting when `Batch()` delegates to the private `boolean()`/`variant()` helpers
- **No new external dependencies** — only stdlib `sync/atomic`
