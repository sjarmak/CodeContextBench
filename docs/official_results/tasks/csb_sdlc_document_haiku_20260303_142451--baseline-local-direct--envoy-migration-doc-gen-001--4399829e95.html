# Migration Guide: Envoy External Processing Filter (v1.30 → v1.31)

## Overview

This guide documents the changes to the Envoy External Processing (ext_proc) HTTP filter when upgrading from **v1.30.0 (April 2024)** to **v1.31.0 (July 2024)**. The ext_proc filter allows external services to process HTTP requests and responses by implementing a gRPC bidirectional stream protocol.

The v1.31 release introduces improvements for route cache management and observability mode features, along with behavioral changes for timeout error handling. Most configurations will work without modification, but recommended updates and new features should be evaluated.

## Breaking Changes

### 1. Timeout Error Status Code Change

**Impact Level:** HIGH - Affects downstream client behavior

#### Change Description
In v1.30, when a message timeout occurred in the external processor, Envoy returned a **`500 Internal Server Error`** status code to downstream clients.

In v1.31, timeout errors now return **`504 Gateway Timeout`** status code.

#### Motivation
The `504 Gateway Timeout` status code is more semantically correct for external service timeouts, allowing clients to distinguish between Envoy-internal errors (500) and upstream/external service timeouts (504). This enables better error handling and retry logic in downstream systems.

#### Migration Impact
- **Downstream clients** may handle 504 responses differently than 500 responses
- **Error handling code** that checks for HTTP status 500 in the context of ext_proc timeouts should be updated
- **Monitoring and alerting** based on status codes should be reviewed
