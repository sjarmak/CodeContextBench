### Core Lifecycle Interface (Scala)
- **`Server` trait** (`core/src/main/scala/kafka/server/Server.scala`) — defines `startup()`, `shutdown()`, `awaitShutdown()`, and the `ProcessStatus` sealed trait (SHUTDOWN → STARTING → STARTED → SHUTTING_DOWN)

### Startup Ordering
`KafkaRaftServer.startup()` starts **controller first**, then **broker** — ensuring controller endpoints are available before broker registration.

### Coordinated Shutdown (Reverse Order)
`BrokerServer.shutdown()` enforces strict reverse-dependency ordering:
1. Stop accepting connections (`socketServer.stopProcessingRequests`)
2. Request handler pool shutdown
3. **Scheduler shutdown early** (prevents background tasks from accessing being-closed resources)
4. Transaction/Group/Share coordinators shutdown
5. ReplicaManager shutdown
6. LogManager shutdown
7. QuotaManagers, then SocketServer (final)
8. LifecycleManager close, then `sharedServer.stopForBroker()`

`KafkaRaftServer.shutdown()` shuts down **broker first** then **controller** (reverse of startup).

### SharedServer Reference Counting
`SharedServer` uses `startForBroker/startForController` and `stopForBroker/stopForController` — shared components only stop when both broker and controller have called stop.

### Java Lifecycle Interfaces
- `Scheduler` interface → `KafkaScheduler` implementation
