**Issue:** The authorization ID validation compares against the wrong variable, breaking delegation token authentication.

**Root Cause:**
- For **password authentication**: `username` = authenticated user, `authorizationId` = same user
- For **delegation token authentication**: `username` = tokenId, `authorizationId` = token owner (different!)

The code at line 132 compares the client-requested authorizationId with `username`, but it should compare with `authorizationId`. This means:
- A token claiming to own "alice" cannot assert authorizationId="alice" 
- The check rejects it because "alice" ≠ "token123" (the tokenId)
- This is an **authentication bypass** that prevents delegation tokens from working

The fix changes one variable in the comparison: from `username` to `authorizationId`.
