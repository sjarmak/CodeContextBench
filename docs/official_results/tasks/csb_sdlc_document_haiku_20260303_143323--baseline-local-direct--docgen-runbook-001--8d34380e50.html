# Prometheus TSDB Compaction Operational Runbook

## 1. Overview

### What is TSDB Compaction?

Prometheus TSDB compaction is the process of merging overlapping time-series database (TSDB) blocks into fewer, larger blocks while removing data marked for deletion (tombstones). Compaction serves several critical purposes:

- **Reduces disk I/O**: Fewer blocks to scan during queries means fewer file descriptor operations
- **Merges overlapping blocks**: Handles cases where multiple blocks have overlapping time ranges
- **Removes deleted data**: Eliminates tombstoned series to reclaim disk space
- **Maintains query performance**: Keeps the block structure optimized for query execution

### TSDB Architecture

Prometheus stores metrics in a hierarchical block structure:

- **Head Block**: In-memory data buffer that accumulates new samples. When it reaches 1.5x the configured chunk range in duration, it becomes eligible for compaction to disk
- **Persisted Blocks**: Time-series data on disk organized by time range, with levels determined by compaction progress
- **Block Levels**: Compaction creates progressively larger blocks:
  - Level 0: Single head block compaction (~2 hours, default)
  - Level 1: Compaction of 2-3 level-0 blocks
  - Level 2: Compaction of 2-3 level-1 blocks
  - And so on...

### Default Block Ranges

By default, Prometheus uses exponential block ranges starting at `MinBlockDuration` (2 hours):
- Level 0: 2 hours
- Level 1: 10 hours
- Level 2: 50 hours
- Level 3: 250 hours
- And more levels based on `MaxBlockDuration`
