## JVM Bytecode Interpreter Implementation

The bytecode interpreter in HotSpot is split across three functional areas:

### 1. Bytecode Dispatch Table
- **`src/hotspot/share/interpreter/templateInterpreter.cpp`** — defines `DispatchTable` and the `_active_table`/`_normal_table`/`_safept_table` globals
- **`src/hotspot/share/interpreter/templateTable.cpp`** — base template table mapping bytecodes to code stubs
- Per-arch **`templateTable_*.cpp`** — bytecode handler implementations for x86, aarch64, arm, ppc, riscv, s390
- Per-arch **`interp_masm_*.cpp`** — `dispatch_next()`/`dispatch_via()` assembly sequences

### 2. Interpreter Template Generation
- **`src/hotspot/share/interpreter/templateInterpreterGenerator.cpp`** — `set_entry_points_for_all_bytes()` populates all dispatch table slots
- Per-arch **`templateInterpreterGenerator_*.cpp`** — emit machine code for each architecture (x86 has three files: shared, 32-bit, 64-bit)

### 3. Method Entry Point Setup
- **`src/hotspot/share/interpreter/abstractInterpreter.cpp`** — `set_entry_for_kind()` registers entries per method kind
- **`src/hotspot/share/interpreter/interpreter.cpp`** — top-level initialization
- `generate_method_entry()` in the template generator dispatches to `generate_normal_entry()`, `generate_native_entry()`, etc.

### 4. Zero (Portable C++) Interpreter
- **`src/hotspot/share/interpreter/zero/bytecodeInterpreter.cpp`** — `BytecodeInterpreter::run()`, the C++ switch-based dispatch loop
- **`src/hotspot/share/interpreter/zero/zeroInterpreterGenerator.cpp`** and **`src/hotspot/cpu/zero/zeroInterpreter_zero.cpp`** — zero-arch specifics
