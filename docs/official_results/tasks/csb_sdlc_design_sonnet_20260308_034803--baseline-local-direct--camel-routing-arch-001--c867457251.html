## Apache Camel Message Routing Architecture

### The Hierarchy

```
Component → Endpoint → Consumer (inbound)
                    → Producer (outbound, via SendProcessor)
Route = Endpoint + CamelInternalProcessor(RoutePipeline([Channel₁→EIP₁, Channel₂→EIP₂, ...]))
```

### Key Design

**Build time** (`RouteReifier.doCreateRoute()`):
1. `RouteDefinition` (DSL model) is walked — each `ProcessorDefinition` node is dispatched to its `ProcessorReifier`
2. Each reifier creates a concrete `Processor` (e.g., `SendProcessor`, `FilterProcessor`)
3. `ProcessorReifier.wrapChannel()` wraps it in a `DefaultChannel` which layers interceptors, tracing/debugger/JMX advices, and error handler
4. All channels are assembled into a `RoutePipeline`
5. The pipeline is wrapped in a root `CamelInternalProcessor` (adds UoW, route policies, JMX)
6. `endpoint.createConsumer(processor)` wires the transport to the full chain

**Runtime** (per message):
```
Consumer → CamelInternalProcessor [UoW, policies]
  → RoutePipeline [iterates via ReactiveExecutor+PipelineTask]
    → DefaultChannel [trace/debug/error handler] → EIP Processor
    → DefaultChannel [trace/debug/error handler] → SendProcessor → Producer
```

### Design Patterns
- **Pipes-and-Filters** (EIP): `Pipeline`/`RoutePipeline`
- **Message Endpoint** (EIP): `Endpoint` + `Consumer`/`Producer`  
- **Decorator/Advice** (GoF): `CamelInternalProcessorAdvice` chain in `CamelInternalProcessor`
- **Template Method** (GoF): `ProcessorReifier.makeProcessor()` → `createProcessor()`
- **Strategy** (GoF): `InterceptStrategy`, `ErrorHandlerFactory`
