# Apache Kafka KafkaConsumer API Reference

## Overview

The `KafkaConsumer` class is the primary client for consuming messages from Apache Kafka. It implements a consumer-group-based messaging pattern where messages are consumed in parallel from multiple partitions of a topic through a coordinating consumer group. The consumer manages offset tracking, partition assignment, and rebalancing automatically or through explicit control.

### Threading Model

**Critical**: `KafkaConsumer` is NOT thread-safe except for `wakeup()`. All methods except `wakeup()` and `close()` must be called from a single consumer thread. The `wakeup()` method can be called safely from any thread and is used to interrupt the consumer thread from blocking operations like `poll()`.

### Key Design Principles

- **Event Loop Model**: The consumer is designed around a single-threaded event loop. The `poll()` method is the central operation that:
  - Fetches records from the broker
  - Manages heartbeats and group coordination
  - Triggers consumer rebalancing
  - Executes rebalance callbacks

- **Lazy Initialization**: Many operations are not performed until the first `poll()` call, which triggers group join and metadata fetching

- **Offset Management**: Consumers track the position (next offset to be read) separately from the committed offset (what has been processed and is safe to skip if restarting)
