# VS Code Diagnostic API Reference

## Overview

The Diagnostic API allows VS Code extensions to report code issues (errors, warnings, information, hints) to users within the editor. Diagnostics are typically generated by language servers, linters, type checkers, or other analysis tools. They provide detailed feedback about problems in source code, including location, severity, message, and related information.

The API is built around the `vscode.languages` namespace and provides mechanisms to:
- Create and manage diagnostic collections
- Report diagnostics for specific files
- Listen to changes in the diagnostic state
- Retrieve current diagnostics across the workspace

**Values:**

| Value | Number | Description |

| `Error` | 0 | Something not allowed by the rules of a language or other means. Errors typically prevent compilation or execution. |
| `Warning` | 1 | Something suspicious but allowed. Warnings indicate potential issues that should be reviewed but don't necessarily prevent execution. |
| `Information` | 2 | Something to inform about but not a problem. Used for informational messages about code patterns. |
| `Hint` | 3 | Something to hint at a better way of doing something, like proposing a refactoring. Hints provide suggestions for improvements. |

**Default Severity:**
The default severity when constructing a `Diagnostic` is `Error` if not specified.
