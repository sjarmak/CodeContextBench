## Bug Report

**Bug ID**: 207835
**Summary**: ACPI video backlight brightness control does not work on Acer TravelMate 5735Z by default

On the Intel GM45/GMA 4500MHD laptop with Debian Sid/unstable and Linux 5.6.7 and GNOME Shell 3.36.2, trying to change the brightness of the internal display using the function keys, GNOME shows the OSD, which seems to have five levels (from acer_wmi?), but the actual brightness does *not* change. There is only `/sys/class/backlight/intel_backlight/brightness` though, and writing values to it seems to work.

Booting the system with `acpi_backlight=vendor`, exposes `/sys/devices/platform/acer-wmi/backlight/acer-wmi/brightness`, but the behavior is the same.

Booting the system with `acpi_backlight=native` or `acpi_backlight=native`, changing the brightness with the function keys works, and there now seem to be 15 (or 16) levels.

**Buggy Function/Data Structure**: `video_detect_dmi_table`

**Confidence**: 0.95

### Root Cause

The Acer TravelMate 5735Z laptop is missing from the DMI (Desktop Management Interface) quirks table in the video backlight detection logic. This causes the system to incorrectly use the ACPI video backlight driver instead of the native backlight driver.

### Analysis

The backlight type detection in `acpi_video_get_backlight_type()` (lines 413-450 of video_detect.c) follows this logic:

1. Check if cmdline option `acpi_backlight=` was provided
2. Check if a DMI quirk matches (from the `video_detect_dmi_table`)
3. Auto-detect based on ACPI capabilities

For the Acer TravelMate 5735Z:
- It has ACPI_VIDEO_BACKLIGHT capability, so it doesn't fall back to vendor driver
- It's not a Windows 8 ready system without a pre-registered native backlight
- No DMI quirk exists, so it defaults to `acpi_backlight_video`

However, the ACPI video backlight interface doesn't work on this hardware. The bug report confirms that booting with `acpi_backlight=native` enables working brightness control, proving that the native backlight driver (intel_backlight from the i915 GPU driver) actually works.

### Required Fix

Add the Acer TravelMate 5735Z to the `video_detect_dmi_table` in `drivers/acpi/video_detect.c` with a callback to `video_detect_force_native()` using DMI matching on `sys_vendor="Acer"` and `product_name="TravelMate 5735Z"`.

The result has been written to `/workspace/fault_localization_result.json`.
