# ExponentialBackoff Function and Backoff Struct

## Function Signature

```go
func ExponentialBackoff(backoff Backoff, condition ConditionFunc) error
```

## Backoff Struct Definition

```go
type Backoff struct {
	// The initial duration.
	Duration time.Duration
	
	// Duration is multiplied by factor each iteration, if factor is not zero
	// and the limits imposed by Steps and Cap have not been reached.
	// Should not be negative.
	// The jitter does not contribute to the updates to the duration parameter.
	Factor float64
	
	// The sleep at each iteration is the duration plus an additional
	// amount chosen uniformly at random from the interval between
	// zero and `jitter*duration`.
	Jitter float64
	
	// The remaining number of iterations in which the duration
	// parameter may change (but progress can be stopped earlier by
	// hitting the cap). If not positive, the duration is not
	// changed. Used for exponential backoff in combination with
	// Factor and Cap.
	Steps int
	
	// A limit on revised values of the duration parameter. If a
	// multiplication by the factor parameter would make the duration
	// exceed the cap then the duration is set to the cap and the
	// steps parameter is set to zero.
	Cap time.Duration
}
```

## Function Behavior

ExponentialBackoff repeats a condition check with exponential backoff. It:

1. Repeatedly checks the condition and then sleeps
2. Uses `backoff.Step()` to determine the length of the sleep and adjust `Duration` and `Steps`
3. Stops and returns when:
   - The condition check returns true or an error
   - `backoff.Steps` checks of the condition have been done
   - A sleep truncated by the cap on duration has been completed

**Returns:** The error from the condition function in case (1), or `ErrWaitTimeout` in all other cases.

```go
type RecoverableError struct {
```

The implementation uses a simple backoff strategy:

```go
const defaultBackoff = 0
```

When no explicit retry timing is provided, operations use zero backoff, meaning immediate retry attempts.

## Retry Duration Parsing

The `retryAfterDuration` function extracts retry timing from HTTP "Retry-After" headers:

```go
func retryAfterDuration(t string) model.Duration {
    parsedDuration, err := time.Parse(http.TimeFormat, t)
    if err == nil {
        s := time.Until(parsedDuration).Seconds()
        return model.Duration(s) * model.Duration(time.Second)
    }
    d, err := strconv.Atoi(t)
    if err != nil {
        return defaultBackoff
    }
    return model.Duration(d) * model.Duration(time.Second)
}
```

This handles both HTTP date-format and numeric second representations of retry delays.

- Network errors from `Client.Do()` calls
- Server errors (5xx status codes)
- Rate-limiting responses (429) when retry-on-rate-limit is enabled

Non-retryable errors (like client errors) are returned directly without wrapping.
