The Problem

AI coding agents fail in production not because the models aren't capable, but because they cannot reliably find and understand the code they need to work with.

Without Code Intelligence

  • Burns 6,000s timeout in Kubernetes monorepo
  • 96 tool calls, 84 minutes, 6 backtracks
  • Finds 2 of 7 files in cross-file refactoring
  • Completes in <2 seconds (never actually ran)
  • Cross-repo tasks are infeasible
vs.

With Code Intelligence

  • Same task completed in 89 seconds
  • 5 targeted searches, 4.4 minutes
  • All 7 files found, 725-line refactoring passes tests
  • 30% cheaper, 100 seconds faster on average
  • Cross-repo discovery is the strongest use case

These are infrastructure problems, not intelligence problems. This book explains the infrastructure.

Will This Topic Still Matter in 3–5 Years?

Problems that persist

  • Codebases are large and getting larger
  • Context windows are finite (even at 1M tokens)
  • Multi-repo reasoning requires infrastructure
  • Retrieval quality ≠ task outcome (fundamental)
  • Agents need governance and evaluation
  • Enterprise deployment needs guardrails

These are structural problems. Better models don't eliminate them — they make the infrastructure layer more important.

What evolves (quarantined)

  • Specific protocols (MCP, etc.) → framed as "current standard"
  • Model names and pricing → callout boxes with dates
  • Tool-specific configs → companion repo, updated continuously
  • Benchmark numbers → baselines, not conclusions

The book teaches systems and patterns. Tool-specific content is modular and replaceable.

The positioning test: "Reliable AI Coding Agents" sounds like a discipline, not a tool guide. It addresses repository scale, context retrieval, and agent evaluation — problems that exist regardless of which models, protocols, or frameworks are current.
